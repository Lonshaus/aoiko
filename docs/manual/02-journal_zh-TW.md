# 02. 建立傳票

手動建傳票、傳票列表瀏覽搜尋、沖銷傳票（修正傳票）。

**Language**: [日本語](02-journal.md) | [English](02-journal_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 手動輸入借方／貸方的多行傳票
> - 在傳票列表按年/月/摘要/金額/交易對象組合篩選
> - 用沖銷傳票把已確定的傳票打掉
>
> **前提**：[01. 初次設定](01-setup_zh-TW.md) 做完（商號・年度，必要的話輔助科目）。

## 1. 手動建傳票 — 首頁畫面

導覽列「**首頁**」畫面的下半段是**新增傳票**表單（`JournalEntryForm`）。

### 1-1. 基本流程

1. **日期**：交易日（預設今天）
2. **摘要**：什麼交易（例：`電費`、`研討會報名費`）
3. 輸入一或多列**借方**：
   - **科目**：下拉選（例：`5130 水道光熱費`）
   - **輔助科目**（選填）：有登記的話（例：`AWS`）
   - **金額**：正整數
   - **稅率**：`10%` / `8%` / `0%`（非課税）
   - **含稅**：勾選＝稅込，取消＝稅抜
4. 輸入一或多列**貸方**（同樣欄位）
5. 「**＋ 新增借方**」 / 「**＋ 新增貸方**」加列（複合傳票）
6. 借方與貸方合計一致會顯示綠色「**✓ 平衡**」徽章。有差額會顯示「**差額 ¥xxx**」警告
7. 按「**新增傳票**」 → 馬上出現在首頁的「最近傳票」列表

### 1-2. 記不動產所得的傳票（僅限設定裡選擇加入時）

在「**設定」→ 使用不動產所得**開啟後，表單上方會出現「**事業」／「不動產**」切換。選「不動產」後科目下拉會只列出不動產用科目（名稱尾巴帶「（不動產）」）。切換狀態會在同一次工作階段內記住，換下一筆傳票不用重新選。

> 沒開啟這個設定時，這個切換完全不會出現，畫面跟原本一樣。

### 1-3. 使用家事分攤

自宅兼事務所的經費要自動拆成「事業使用分 ＋ 事業主貸」時用。

1. 該列勾「**家事分攤**」
2. 填比例（例：`0.30` = 事業使用 30%、剩下 70% 自動進事業主貸）
3. 按新增時這列會自動展開成 2 列（事業經費 + 1610 事業主貸）

> 預設比例可在「**設定」→ 家事分攤預設比例**登記（每列可覆寫）。

### 1-4. 清除

打到一半想全部重來、按「**清除**」。

### 1-5. 記錄品項與數量（簡易庫存管理）

選擇**仕入（5020）**或**売上高（4110）**這兩個科目的行時，會出現**品項**下拉選單跟**數量**欄位。品項主檔可在「**設定**」裡新增/刪除。記錄之後，[06. 報表](06-reports_zh-TW.md)的損益計算書會顯示依法定預設「最終仕入原價法」試算出的**期末庫存估算金額**（僅供參考，不會自動記帳）。

> 如果已申報使用最終仕入原價法以外的評價方法，勾選「**設定」→ 評價方法非最終仕入原価法**即可關閉這個自動試算，回到手動記期末庫存的做法。

### 1-6. 加上部門標籤

摘要右邊的「**部門標籤**」欄可以自由輸入標籤（例：`台北店`、`網路銷售`），用來輕量區分多個據點、事業線或通路。輸入過的標籤會出現在自動完成建議裡。

> 部門標籤只是集計、篩選用的顯示標籤，完全不影響稅額計算或`.xtx`輸出。可以在[06. 報表](06-reports_zh-TW.md)的科目明細裡依部門篩選。

### 1-7. 附加憑證照片

摘要下方的「**憑證照片**」欄選擇照片後，會跳出附有預覽圖的確認對話框。確認內容無誤後按「**附加**」，傳票確定的同時照片也會一起儲存。要附加多張照片就重複選擇（送出前可以用縮圖上的「×」取消）。

> **傳票確定後無法替換或刪除**附件（電子帳簿保存法要求的真實性保障）。附錯照片的話請走訂正仕訳處理，原本的照片會保留在歷史紀錄裡。
>
> 確認對話框可以在「**設定」→ 略過附加憑證照片前的確認**裡跳過（之後隨時可以再勾回來）。

### 1-8. 記錄外幣交易

aoiko沒有專門的多幣別功能或匯率查詢。實務上多數外幣交易在要記帳的當下，日圓金額其實已經確定了——信用卡對帳單、銀行匯款水單等單據，都是照發卡機構/銀行自己的清算匯率換算好的，直接把那個日圓金額當一般傳票記就好（在摘要備註外幣金額，例如`USD 100`，方便日後對照）。

如果是還沒結清的外幣応收/応付帳款，或是在國外現金採購需要自己估算日圓金額的情況，就用當下自己判斷合理的日圓金額照常記帳。等實際結清金額確定後，把差額用「為替差損」「為替差益」（在「**設定」→ 科目**新增）另開一筆傳票調整。

> **為什麼沒有專屬功能**：freee跟Money Forward的個人事業主方案標準功能都沒有自動匯率查詢、自動計算為替差損益（freee只有另外付費的外掛才有這個功能）。實務上「發生當下的金額照記，結清時的差額用為替差損益調整」這樣的做法就已經足夠，aoiko採用一樣的方針。

### 1-9. 記錄員工薪資與扣繳稅額

aoiko沒有薪資計算或自動扣繳稅額功能。如果你聘僱一般員工（不是青色申告的専従者/家族從業員），扣繳稅額請自己用國稅廳公告的「[給与所得の源泉徴収税額表](https://www.nta.go.jp/publication/pamph/gensen/zeigakuhyo2026/01.htm)」算好，用一般傳票記帳。

1. **借方**：`5230 給料賃金`記給與總額（社會保險費扣除前）
2. **貸方**：`1130 普通預金`等記實際匯款的手取金額
3. **貸方**：`2xxx 預り金`（沒有的話在「**設定」→ 科目**新增）記扣繳稅額＋社會保險費本人負擔部分

> **為什麼沒有專屬功能**：連freee個人事業主版的「確定申告」都沒有做扣繳稅額計算、按月試算——使用者只是把確定金額填進彙總欄位，實際計算是另一個產品「freee人事労務」在做。個人事業主聘僱一般員工這種情境本身相對少見，aoiko採用一樣的方針。

### 常見傳票範例

| 交易 | 借方 | 貸方 |
|---|---|---|
| 銷貨用信用卡收 | 1320 未収入金 | 4110 売上高 |
| 經費刷卡 | 5xxx 各經費 | 2120 未払金 |
| 卡費從銀行自動扣款 | 2120 未払金 | 1130 普通預金 |
| 現金投入事業 | 1110 現金 | 3110 元入金 |
| 從事業帳戶領出私用 | 1610 事業主貸 | 1130 普通預金 |
| 自宅租金事業分（家事分攤 30%）| 5260 地代家賃 30% / 1610 事業主貸 70% | 1130 普通預金 |

## 2. 看傳票 — 傳票列表

導覽列「**傳票列表**」進入 `JournalList` 畫面。

### 2-1. 篩選器

畫面上方一排篩選器：

| 篩選器 | 輸入 |
|---|---|
| **年** | 數字（例：`2026`）|
| **月** | 下拉（`全部月份` / `1`〜`12`）|
| **摘要含字串** | 文字（部分符合，Enter 或失焦套用）|
| **金額（下限）** | 數字，留空＝不限 |
| **金額（上限）** | 數字，留空＝不限 |
| **交易對象** | 下拉（從已登記的交易對象）|

> 支援**兩個以上任意組合**篩選，符合電子帳簿保存法的「優良な電子帳簿」検索機能要件。

按「**重設**」清空（年回到今年、月回到本月）。

### 2-2. 看傳票明細

點傳票列展開。借方／貸方各明細（科目・輔助科目・稅率・金額・摘要）會顯示出來。`▾` / `▸` 標示展開狀態。

展開的明細下方會列出**憑證照片**（點縮圖看原尺寸），還有補加照片的欄位。不管這筆傳票是CSV匯入、OCR掃描還是手動輸入的，都可以在這裡事後補上憑證照片（確認流程跟1-7一樣）。

### 2-3. 分頁

50 筆一頁。下方「**← 前**」 / 「**次 →**」換頁。

## 3. 沖銷傳票（修正傳票） — 改錯

> aoiko **不允許直接編輯已確定的傳票**。為符合電子帳簿保存法的訂正履歷保持要件，錯誤透過**沖銷傳票**（reverse entry）修正。原傳票會保留為歷史紀錄。

### 3-1. 沖銷步驟

1. 在傳票列表點該傳票展開
2. 按「**沖銷**」按鈕
3. 確認對話框：「建立把借方貸方互換的沖銷傳票（今天日期），原傳票標記為『已沖銷』」→ 「**沖銷**」
4. 原傳票顯示「**已沖銷**」徽章＆刪除線
5. 新的沖銷傳票出現在列表（今天日期）
6. 用正確內容重新建傳票（手動／CSV／OCR／訂單取込皆可）

### 3-2. 沖銷例

原傳票：
```
2026-05-01  電費
  借方  5130 水道光熱費  8,000
  貸方  1130 普通預金    8,000
```

按「沖銷」會自動產生今日的沖銷傳票：
```
2026-05-26  沖銷：電費
  借方  1130 普通預金    8,000
  貸方  5130 水道光熱費  8,000
```

帳本上合計變 0。再用正確金額（例 8,500）建一張新傳票。

### 3-3. 已沖銷的能取消嗎

一次沖銷後就不能再對該傳票按「沖銷」（「**已沖銷**」徽章出現，按鈕不顯示）。對沖銷傳票本身再做沖銷可以實質還原原狀、但歷史會疊起來。

> CSV 匯入導致的大量錯誤建議從**匯入紀錄**用「**整批沖銷**」處理（[03. CSV 匯入](03-csv-import_zh-TW.md)）更有效率。
