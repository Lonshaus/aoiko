# 11. 備份與還原

File System Access API、OPFS、手動 zip 下載／還原的運用。

**Language**: [日本語](11-backup.md) | [English](11-backup_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 指定備份資料夾、開自動備份
> - 手動匯出備份檔案
> - 從備份還原（含憑證照片）
> - 知道資料消失風險跟對策
>
> **前提**：[01. 初次設定](01-setup_zh-TW.md) 做完、資料已累積記帳中。

## 1. 為什麼要備份

aoiko 的資料存在瀏覽器的 **IndexedDB**（本機）。這代表：

- ✅ 不經伺服器、不外洩（隱私）
- ❌ 瀏覽器站點資料清除 → **完全消失**
- ❌ 裝置故障 → 還原不能
- ❌ 瀏覽器 profile 刪除 → 失去

定期備份是 **利用者責任**。「想到再備份」的運用是事故的開始、推薦自動備份。

## 2. 備份方式比較

| 方式 | API | 對應瀏覽器 | 推薦度 |
|---|---|---|---|
| **File System Access API（FSA）** | `showDirectoryPicker` | Chrome / Edge / Brave 等 Chromium 系 | ◎ 自動・任意資料夾 |
| **OPFS（Origin Private File System）** | `navigator.storage.getDirectory` | Safari / Firefox 含大部分 | ◯ 自動但是瀏覽器管理區 |
| **手動 JSON 下載** | `<a download>` | 全瀏覽器 | △ 想到才備 |

aoiko 自動 fallback：FSA 可用就用 FSA、不行 OPFS、再不行只剩手動下載。

## 3. 設定自動備份（推薦）

設定 → 「**備份**」區塊。

### 3-1. Chromium 系（FSA 對應）

1. 點 「**選擇備份資料夾**」
2. 瀏覽器跳資料夾選擇對話框
3. **隨便選一個地方**（例：`Documents/aoiko-backup/`、Google Drive 同步資料夾、Dropbox 內 等）
4. 存取權限對話框按 「**允許**」
5. 設定畫面顯示 「**目前的資料夾：〇〇**」 即成功

之後每次傳票新增・修改、會自動寫 `aoiko-ledger-{年月日}.zip` 形式的備份。zip 裡除了帳本資料（JSON）之外，還同梱憑證照片原圖（[02. § 1-7](02-journal_zh-TW.md#1-7-附加憑證照片)）。

> **Google Drive / iCloud / Dropbox 連動 tip**：用 FSA 指定的本機資料夾若是雲端同步對象、實質等於有雲端備份。例：`~/Google Drive/My Drive/aoiko-backup/` → 本機寫的 JSON 自動同步到 Google Drive。
>
> **注意**：iCloud Drive Finder「Download On Demand」項目當 FSA 資料夾、每次備份寫入都觸發線上同步、可能不穩。選本機有空間的地方比較好。

### 3-2. Safari / Firefox（只能 OPFS）

不支援 FSA 的瀏覽器只能用 **OPFS**。OPFS 是 **瀏覽器內部管理的私有 storage**、Finder 跟檔案總管直接看不到。

OPFS 備份：
- 自動寫入（跟 FSA 一樣每次傳票更新都寫）
- 瀏覽器站點資料清除會把 **OPFS 一起清掉**（跟 IndexedDB 同命運）
- 備份「裝置/瀏覽器獨立」的本來目的 **達不到**

> Safari / Firefox 用戶強烈建議跟 **手動 JSON 下載** 併用。

### 3-3. 最終備份時刻確認

設定 →「備份」區塊顯示 「**最終備份：2026-05-26 14:23**」等。長時間沒更新就用手動匯出補。

## 4. 手動匯出

設定 → 「**備份**」區塊 → 「**下載備份**」：

- 全部資料（傳票、輔助科目、交易對象、固定資產、設定、憑證照片 等）匯到 1 個 zip 檔
- 存到瀏覽器「下載」資料夾
- 檔名：`aoiko-ledger-{時間戳}.zip` 等

> **API 金鑰・申報者資訊預設不會納入**。除非你把「備份時包含 API 金鑰」「納入申報者資訊」這兩個開關打開，否則不會有明文 API 金鑰或個人資訊被寫進雲端同步資料夾。只有要換到另一台裝置、想連同這些一起帶走時才需要打開。

把這個檔案：

- 複製到別的物理 storage（外接 SSD、USB）
- email 寄給自己
- 存到雲端 storage

…等做 **多處分散**保存比較穩。

> 月末・季末・年度末等 **節點手動下載** 的習慣、加上自動備份就完整。

## 5. 從備份還原

### 5-1. 還原情境

- 不小心清掉瀏覽器快取
- 要換到新 PC
- 換瀏覽器（Chrome → Safari 等）

### 5-2. 還原步驟

1. 設定 → 「**從備份還原**」區塊
2. 「**選擇檔案**」指定 zip（新格式）或 JSON（舊格式）——會依副檔名／內容自動判定，只有一個按鈕
3. 內容摘要顯示：
   > version 1 ・ 12 個表格 ・ 5,432 行
   > 包含 38 張憑證照片（只有zip且含照片時才顯示）
4. 點 「**全資料置換還原**」
5. 確認對話框：
   > 要置換全資料嗎？
   > 目前的全資料會被刪除、被選檔案的內容完全置換。此操作不能還原。
6. 「**置換還原**」執行
7. 還原成功訊息 → 「**重新載入**」 App reload

### 5-3. 還原時注意

- **完全置換**：目前 IndexedDB 整個被覆蓋。誤操作沒救
- **進行中的工作先手動匯出** 再還原是好習慣
- 還原後必跑報表的 **BS 一致性確認**（[06. § 4-1](06-reports_zh-TW.md#4-1-不一致警告)）
- **還原舊格式（純 JSON）備份不會還原憑證照片**（舊備份本來就沒有照片）。傳票、金額等帳本資料本身照常正確還原

## 6. 全資料刪除（謹慎）

設定 →「資料管理」→ 「**全資料刪除**」：

- IndexedDB 全部資料物理刪除（備份檔案保留）
- 警告對話框後執行
- 刪除後回初期狀態、要重新同意免責・輸入基本資訊

> **什麼時候用？**：要乾淨重啟用測試資料、把 aoiko 轉讓 等限定場合。不常用。

## 7. 推薦備份策略

3 層構成：

| 層 | 目的 | 實裝 |
|---|---|---|
| **層 1**：常時自動 | 短期保護（不小心操作的對應）| FSA 指定雲端同步對象資料夾 |
| **層 2**：節點手動 | 中期保護（月末・年度末 snapshot）| 手動 JSON 匯出 → 別處保存 |
| **層 3**：定期確認 | 健全性確認 | 半年一次、在別瀏覽器 profile 跑還原測試 |

> 申告完的年度特別重要。年度鎖定（[06. § 8](06-reports_zh-TW.md#8-年度鎖定申告済み)）併用、變更也能偵測。

## 8. 交接資料給稅理士

設定 → 「**稅理士資料匯出**」 區塊，可以匯出仕訳資料的 CSV，交接給稅理士（跟備份 zip 是不同的東西——備份是用來還原 aoiko 本身的全部資料，這個是用來匯入其他會計軟體用的）。

| 檔案 | 格式 | 用途 |
|---|---|---|
| 弥生形式CSV | Shift-JIS、CRLF、25 欄 | **預設・推薦**。弥生会計、freee、Money Forward 等多數會計軟體都有「弥生形式匯入」功能可以讀取 |
| 通用CSV | UTF-8、每個仕訳明細一行 | 弥生形式無法使用時的備用選項 |
| 訂正履歷（取消履歷）CSV | UTF-8 | 訂正仕訳（原仕訳・打消し仕訳）清單。主要兩個CSV都會把訂正配對從統計中排除，需要另外確認被取消了什麼時使用 |

對象年度是設定裡「基本資訊」目前選擇的年度。

> **關於稅區分**：弥生形式CSV的「稅區分」欄位（課稅銷售・課稅進貨等）是aoiko自行推算輸出的，並非官方保證正確的規格，尤其是輕減稅率、發票扣除比例、簡易課稅事業區分等部分，匯入前請務必請會計事務所自行核對。
