# 12. 修正申告

申告済年度 snapshot 的建立、差分偵測、修正傳票的做法、提交手順。

**Language**: [日本語](12-amended.md) | [English](12-amended_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 把申告済的年度鎖定、保留「申告當下的 snapshot」
> - 之後發現錯誤時、看差分、加修正傳票
> - 整理修正申告書提交需要的資訊
>
> **前提**：[06. 報表](06-reports_zh-TW.md) 該年度已鎖定（[§ 8](06-reports_zh-TW.md#8-年度鎖定申告済み)）。

## 1. 什麼是修正申告

確定申告書提交**之後**才發現税額計算或所得計算有錯：

- **税額申告太少**（要補繳）→ **修正申告**
- **税額申告太多**（可退稅）→ **更正の請求**

兩者的提交様式・處理在 e-Tax 側不同。aoiko 側的作業兩種都差不多（改資料、看差分）、但 e-Tax 側手續不同。

> 本工具不直接支援申告書的建立・提交。只提供差分偵測跟「什麼變了」的整理。

## 2. 申告済 snapshot

[06. 報表 § 8](06-reports_zh-TW.md#8-年度鎖定申告済み) 按 「**鎖定為申告済み**」 時、aoiko 把當下的：

- 各勘定科目別合計（PL・BS 全項目）
- 主要報表（月別銷貨、固定資產明細 等）

…存成 **snapshot** 到 IndexedDB。這就是申告書提交時寫的內容的 snapshot。

> 申告書原本（紙 or e-Tax 提交資料的存檔）**請務必另外保存**。aoiko snapshot 是數值 base、申告書本體（個人資訊・所得控除 等）沒涵蓋。

## 3. 修正工作流

### 3-1. 解除鎖定

要加設定變更必須先解鎖。

1. 報表 → 選該年度
2. 🔒 申告済み徽章旁邊按 「**解除鎖定**」
3. 確認後年度變解鎖狀態

> snapshot **不會刪、保留**（重要）。

### 3-2. 加入差分對象

解鎖後：

- 傳票 **追加**（漏的經費、漏申告的銷貨 等）→ 照常用 [02. 建立傳票](02-journal_zh-TW.md)
- 傳票 **訂正**（金額或科目錯）→ [02. § 3 沖銷傳票](02-journal_zh-TW.md#3-沖銷傳票修正傳票--改錯) 沖銷 → 建立正確新傳票
- 固定資產 **重新登錄**（耐用年數修正、廃棄處理 等）→ [08. 減價償卻](08-depreciation_zh-TW.md)

> 修正傳票的 `description` 寫清楚（例：「2026 年分修正申告 — 銷貨漏申告追加（XXX 公司）」）。之後差分原因要看得出來。

### 3-3. 差分確認

加入修正傳票後、報表 → **概況**區塊會：

- 顯示目前值（修正後）與 snapshot 的**差**
- 各項目 `Before → After` 並排
- 主要變化（PL 所得增減、BS 殘高變動）摘要

> 差分跟想像一致嗎？務必確認。預期外的地方變了的話、檢查相關傳票。

### 3-4. e-Tax 側建立修正申告書

aoiko 上數字固定後：

1. 開啟 e-Tax 軟體（下載版） 或 確定申告書等作成コーナー（所得稅的修正申告 WEB 版不支援）
2. 「**修正申告書作成**」 選單（正式名稱依當年度 e-Tax UI 而定）
3. **當初申告（決算書〔青色申告決算書／収支内訳書〕→ 確定申告書）的值**：把 aoiko 再次輸出的 `.xtx` 組み込み到下載版，或在作成コーナー手動輸入
4. 「**正確的金額**」 用修正後數字輸入
5. 差分計算 e-Tax 側自動。延滯稅・加算稅概算也會顯示
6. 提交（電子）

> aoiko 的 `.xtx` 再次輸出（[10. `.xtx` 輸出](10-xtx-export_zh-TW.md)）也反映修正後的決算書、可作為手動輸入的參考。

### 3-5. 提交後重新鎖定

提交修正申告書之後：

1. 報表 → 該年度 → 「**鎖定為申告済み**」 重新鎖
2. **用新 snapshot 覆蓋**（修正後狀態成為新的「申告済み」值）

> 舊 snapshot 會失去。「修正申告前 vs 修正申告後」事後就比不了、申告書原本（提交資料）才是真實紀錄。

## 4. 常見情境

### 情境 A：銷貨計上漏

```
發現：2026 年分確定申告提交後、12 月份銷貨 100,000 円漏了
```

1. 解鎖
2. 加傳票：`借方 1320 未収入金 100,000 / 貸方 4110 売上高 100,000`（日期 2025-12-31 等）
3. 確認差分：所得 +100,000
4. e-Tax 修正申告 → 補繳
5. 重新鎖定

### 情境 B：經費二重計上

```
發現：同樣的 AWS 費用 5,000 円、被 CSV 取込跟手動傳票各登錄了一次
```

1. 解鎖
2. 從 [03. CSV § 5](03-csv-import_zh-TW.md#5-匯入紀錄與整批沖銷) 沖銷該 batch、或把手動那筆沖銷
3. 確認差分：經費 -5,000、所得 +5,000
4. e-Tax 修正申告 → 補繳
5. 重新鎖定

### 情境 C：減價償卻耐用年數錯誤

```
發現：原本以 4 年攤的 MacBook、其實 4 年沒錯但經費計上有漏
```

1. 解鎖
2. 固定資產 → 該資產確認 → 取得日・耐用年數・科目重新檢查
3. 既存的減價償卻傳票沖銷（reverse → 用正確計算重新生成）
4. 確認差分
5. e-Tax 修正申告
6. 重新鎖定

### 情境 D：所得控除修正（aoiko 範圍外）

```
發現：醫療費控除忘了報
```

aoiko 不處理所得控除。在 e-Tax 側直接做「**更正の請求**」。aoiko 上不用動（事業所得側沒變化）。

## 5. 期限與注意

### 修正申告提交期限

- **修正申告**（補繳）：隨時可。發現越早交越好（延滯稅天天增）
- **更正の請求**（退稅）：原則從申告期限算起 **5 年內**

### 加算稅・延滯稅

- 修正申告會被加 **過少申告加算稅**（10% 或 15%）跟 **延滯稅**（年率變動）
- 在稅務署調查通知前自願修正、加算稅可減免甚至免除
- 細節跟稅理士・稅務署確認

### 帳簿保存

- 修正申告後的 **修正傳票** 電子帳簿保存法上保存 7 年
- 修正前後的差分在 aoiko 內以「沖銷傳票履歷」形式保留、也要保存
- 用備份（[11. 備份與還原](11-backup_zh-TW.md)）好好保管
