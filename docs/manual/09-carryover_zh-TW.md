# 09. 前期繰越

年度切換時，把前年度末殘高當作當年度的期首振替傳票繰越過來。

**Language**: [日本語](09-carryover.md) | [English](09-carryover_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 年度最初把前年度末殘高建成期首振替傳票
> - 知道純利益・事業主貸借被元入金吸收的邏輯
> - 已建立的繰越傳票可以刪除、重建
>
> **前提**：前年度的傳票登錄完、年度末鎖定 or 大致確定的狀態。

## 1. 什麼是前期繰越

aoiko 不論青色申告或白色申告都用複式簿記記帳，所以每年都會做 **損益計算書**（PL）跟 **資產負債表**（BS）。BS 是年度末時點的殘高、**翌年度從那個殘高開始**、所以新年度初要：

- 前年度末的**資產殘高**當新年度的借方
- 前年度末的**負債殘高**當新年度的貸方
- 前年度的**純利益**跟**事業主貸借**吸收到**元入金**

…這樣的期首振替傳票是必要的。

aoiko 用 「**設定 → 前期繰越（期首殘高）**」 自動產生。

## 2. 期首振替傳票的中身

例：2026 年度的期首振替（前年 2025 年末時點）

```
2026-01-01  前期繰越（期首殘高）
  借方  1110 現金             500,000
  借方  1130 普通預金        1,200,000
  借方  1310 売掛金           300,000
  借方  1514 工具器具備品     250,000    (取得價額 - 累積償卻)
  貸方  2120 未払金            80,000
  貸方  3110 元入金         2,170,000    (= 借方合計 − 其他貸方)
```

### 元入金計算式

`新年度元入金 = 前年末元入金 + 前年純利益 + 事業主借 − 事業主貸`

也就是：
- **前年純利益** 吸收到元入金（黑字的話元入金增）
- **事業主貸**（生活費領出）從元入金扣
- **事業主借**（個人資金投入）加到元入金

> 這樣新年度元入金就代表「事業的純資產」。事業主貸借勘定 reset、新年度又從 0 開始累積。

## 3. 期首振替步驟

### 3-1. 選當年度

設定 → 「**處理中的年度**」 選當年度（例 2026）→ 「**儲存**」。

### 3-2. 預覽

設定 → 「**前期繰越（期首殘高）**」 區塊按 「**預覽**」：

- 借方（資產）和貸方（負債・元入金）的內訳會顯示
- 元入金計算根據也一起：
  - **前年末元入金**：前年的繰越傳票的元入金（或前前年繰越過來的）
  - **前年純利益**：前年度 PL 的純利益
  - **事業主貸借**：前年度的事業主貸（withdrawals）/ 事業主借（contributions）

預覽看內容對不對。

### 3-3. 建立

沒問題就按 「**建立期首振替傳票**」。`{年}-01-01` 日期的期首振替傳票 1 張建立、首頁「最近的傳票」與報表的 BS 會反映。

成功訊息：
> ✓ 已建立期首振替傳票

### 3-4. 重做（刪除後重建）

有什麼錯誤的話：

1. 「**刪除既存的繰越傳票**」 把 `{年}-01-01` 的繰越傳票 1 件刪除（**物理刪除而非沖銷傳票**：期首振替不是會計實態認識而是機械產生的 meta data 性存在、不在訂正履歷保持要件範圍）
2. 修前年度的傳票・固定資產
3. 再 「**預覽**」 → 「**建立期首振替傳票**」

### 3-5. 錯誤

| 錯誤訊息 | 意思 |
|---|---|
| 已存在前期繰越傳票。請先刪除。 | 已經有 `{年}-01-01` 日期的繰越傳票。看 3-4 |
| 前年沒有傳票故無法繰越 | 前年度 0 件傳票（初年度等）|

## 4. 初年度（前年度沒傳票時）

第一次用 aoiko 的第一個年度沒前年資料、會出 「**前年沒有傳票故無法繰越**」。這時不要用前期繰越，改用開業專用的畫面：

- 如果有開業前購入、開業後才開始用的資產（轉用資產），或是要登記開業費，用 **[13. 開業時設定（開業精靈）](13-opening-setup_zh-TW.md)**，可以自動算出未償卻殘額、自動生成對應元入金的傳票
- 如果只是單純投入現金或已經知道帳面價值的資產，也可以手動建傳票：

```
2026-01-01  開業時殘高
  借方  1110 現金             500,000
  借方  1514 工具器具備品     200,000
  貸方  3110 元入金           700,000
```

[02. 建立傳票 § 1](02-journal_zh-TW.md#1-手動建傳票--首頁畫面) 的手順手動輸入。

> 開業屆提交日跟業務用 PC 取得日的稅務處理建議跟稅理士確認。

## 5. 跨年度的工作流

1. **年度末**（例 2025-12-31）前把前年度傳票做完
2. 報表 → 「**鎖定為申告済み**」 鎖定 2025 年度（[06. § 8](06-reports_zh-TW.md#8-年度鎖定申告済み)）
3. 設定 → 「**處理中的年度**」 改 2026
4. 設定 → 前期繰越 → 「**建立期首振替傳票**」
5. 開始記 2026 年度的帳

> 注意：固定資產的**當年度償卻傳票** 跟前期繰越不同。前年度末以前產生減價償卻傳票（[08. § 3](08-depreciation_zh-TW.md#3-年末產生減價償卻傳票)）→ 然後前期繰越 才是正確順序。

## 6. 注意

- 期首振替傳票用 `2026-01-01` 這種**年初日**建
- 同年度想建第 2 次會跳錯誤（重建用刪除→建立）
- 期首振替傳票的 `description` 是「前期繰越（期首殘高）」
- 內部標記 `source: 'carryover'`（[02. § 2-1](02-journal_zh-TW.md#2-1-篩選器) 可用 source 搜尋）
