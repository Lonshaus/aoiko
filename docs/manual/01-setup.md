# 01. 初回設定

**Language**: **日本語** | [English](01-setup_en.md) | [繁體中文](01-setup_zh-TW.md)

aoiko を起動してから記帳を始められる状態にするまでの手順。

> **この章を読み終えたら出来ること**
> - 免責事項に同意し、屋号・年度・消費税方式を登録できる
> - `.xtx` 提出に必要な申告者情報・青色申告特別控除を登録できる
> - 補助科目（口座区別など）と取引先を登録できる
> - OCR/LLM を使う場合の API キー／エンドポイントを設定できる
>
> **前提**：[メイン README](../../README.md) の手順で aoiko を起動済み、ブラウザで開けている状態。

## 1. 免責事項に同意する

起動直後、画面中央に **免責事項ダイアログ**が表示されます。aoiko が出力する数値・帳簿の正確性は保証されません。実申告での利用は税理士・税務署に必ず確認してください。

内容を確認の上、**「同意する」** をクリックすると本体画面に進めます。同意は IndexedDB に記録され、再表示は基本ありません（免責文の重要改訂時のみ再要求）。

> 詳細は [DISCLAIMER.md](../../DISCLAIMER.md) を参照。

## 2. 言語を切り替える（任意）

ナビ右の **「設定」** をクリック → 一番上の **「言語」** セクションで `日本語 / English / 繁體中文` を選択 → **「保存」**。

> 既定は日本語。本ガイドは日本語 UI 基準で書かれています。

## 3. 基本情報を登録する

設定画面の **「基本情報」** セクションを開きます。決算書・確定申告書 B・`.xtx` ファイルに記載される事業者情報です。

| 項目 | 入力する内容 | 例 |
|---|---|---|
| 事業名 / 屋号 | 確定申告書の屋号欄に出る名前 | `青井ウェブ事務所` |
| インボイス登録番号 | 適格請求書発行事業者登録番号 | `T1234567890123` |
| 処理中の年度 | 帳簿を付ける対象年度 | `2026` |

入力後 **「保存」** を押す。インボイス未登録なら登録番号は空欄のままで OK。

## 3a. 申告者情報・青色申告特別控除を登録する（.xtx 提出に必須）

`.xtx` を e-Tax ソフト(ダウンロード版) に組み込むために必須の情報です。設定画面の **「申告者情報」** セクションで登録します。

| 項目 | 入力する内容 |
|---|---|
| 提出先税務署 | 署名またはコードで検索して選択（5 桁） |
| 利用者識別番号 | e-Tax の 16 桁の番号 |
| 氏名・名称 | 確定申告書に記載する氏名 |
| 郵便番号（任意） | ハイフン無し 7 桁 |
| 住所 | 確定申告書に記載する住所 |

> **本機にのみ保存され、既定ではバックアップに含まれません**（[11. バックアップと復元](11-backup.md) 参照）。

続けて **「青色申告特別控除」** セクションで **「控除区分」** を選択：

- **正規の簿記＋e-Tax／優良な電子帳簿（65 万円）**：複式簿記 + e-Tax 提出 + 優良な電子帳簿保存の要件を満たす場合
- **正規の簿記のみ（55 万円）**：複式簿記のみ、e-Tax 提出や優良な電子帳簿の要件を満たさない場合
- **簡易簿記（10 万円）**
- **控除なし**

> 75 万円控除は令和 9 年分以降、上記に加えて期限内の e-Tax 提出が必須です。詳細は [07. 消費税](07-consumption-tax.md) や決算書の控除額表示で確認してください。

## 4. 消費税方式を選ぶ

**「消費税」** セクションで、課税方式を選択します（本ツールは消費税の **概算・比較のみ**、実申告書の出力は対象外）。

### 4-1. 納税義務

- **課税事業者**：消費税申告が必要。下の課税方式も選択
- **免税事業者**：消費税申告不要（売上 1,000 万以下 & インボイス未登録）

### 4-2. 課税方式（課税事業者の場合）

| 方式 | 向いているケース |
|---|---|
| 本則課税 | 仕入れが多く、仕入税額控除を最大限活用したい |
| 簡易課税 | 売上 5,000 万以下、仕入が比較的少ない、計算を楽にしたい |
| 2 割特例（2023/10〜2026/9） | インボイス開始に伴う 3 年限定の負担軽減措置 |
| 3 割特例（令和 9・10） | 売上 5,000 万以下向けの新設特例 |

簡易課税を選んだ場合は **「簡易課税 事業区分」** で第 1〜6 種を選択：

- 第 1 種：卸売業
- 第 2 種：小売業、農林漁業（飲食料品）
- 第 3 種：製造業・建設業・農林漁業（その他）
- 第 4 種：その他（飲食店業など）
- 第 5 種：サービス・金融・運輸通信
- 第 6 種：不動産業

**「保存」** を押す。

> 方式の詳しい選び方は [07. 消費税](07-consumption-tax.md) で扱います。迷ったら本則課税で進めて、年末に「レポート」画面の **消費税 4 方式比較** で実数を見比べる方法もあります。

## 5. 補助科目を登録する（口座・経費の細目）

複数の銀行口座を持っている／同じ経費科目で取引先別に分けたい場合に使います。

設定画面の **「補助科目」** セクションで：

1. **「親科目」** ドロップダウンから科目を選択（例：`1130 普通預金`）
2. **「補助科目名」** に入力（例：`三菱UFJ 本店`）
3. **「追加」** をクリック

よくある登録例：

| 親科目 | 補助科目名 | 用途 |
|---|---|---|
| 1130 普通預金 | 三菱UFJ 本店 | 口座別の残高管理 |
| 1130 普通預金 | SBI新生 | 同上 |
| 2120 未払金 | 楽天カード | カード別の未払金管理 |
| 2120 未払金 | au PAY カード | 同上 |
| 5150 通信費 | AWS | 経費の細分 |
| 5150 通信費 | 携帯 | 同上 |

> CSV インポート時の **既知側科目** はここで登録した補助科目を選択できます（例：「楽天カード CSV → 2120 未払金 / 楽天カード」）。

## 6. 取引先を登録する

インボイス登録番号や既定の対方科目を取引先と紐づけておくと、CSV インポート時に **自動分類ルール**から「この取引先 → この科目」を自動適用できるようになります。

設定画面の **「取引先」** セクションで：

1. **「取引先名」** を入力
2. **「区分」** を選択：法人 / 個人 / 公共 / 海外 / 不明
3. **「インボイス番号」**（任意）：`T` + 13 桁
4. **「既定の対方科目」**（任意）：その取引先で多い経費科目（例：`AWS → 5150 通信費`）
5. **「追加」** をクリック

> 取引先別の集計や仕訳一覧での絞り込みもここで登録した取引先を参照します。

## 7. OCR/LLM を使う場合の準備

OCR（領収書）・LLM 分類（CSV 自動分類）・注文取込（Amazon / 楽天 等の貼り付け解析）を使いたい場合のみ必要。使わないなら飛ばして OK。

設定画面の **「LLM 連携」** セクションで、**「OCR/LLM エンジン」** を選択：

### 7-A. Google Gemini（既定・クラウド）

無料枠あり、設定が一番簡単。

1. [Google AI Studio](https://aistudio.google.com/apikey) で API キーを取得（Google アカウントが必要）
2. aoiko の **「Gemini API キー」** に貼り付け
3. **「保存」** → **「接続テスト」** で `✓ 接続成功` を確認

> **注意**：CSV 行・領収書画像が Google のサーバに送信されます。送信前に **確認ダイアログ**が出ます。Google のプライバシーポリシーに従って取り扱われ、無料枠は学習利用される場合があります。詳細は [PRIVACY.md](../../PRIVACY.md)。

### 7-B. OpenAI 互換 / Ollama 等（ローカル可）

Ollama / LM Studio / llama.cpp / vLLM などを自分の PC や任意サーバで動かしている場合。

1. エンジン選択で **「OpenAI 互換 / Ollama 等」**
2. **エンドポイント（baseURL）** を入力：例 `http://localhost:11434/v1`
3. **API キー** を入力（Ollama 等ローカルでは通常不要、空欄可）
4. **「モデル一覧取得」** をクリック → OCR 用モデル / 分類用モデルをそれぞれ選択
5. **「保存」** → **「接続テスト」**

> - OCR には **vision 対応モデル**が必須（llama3.2-vision / ministral-3 / moondream 等）
> - localhost を指定する場合、aoiko も **ローカル実行版**（`npm run preview`）である必要があります（HTTPS で配信された aoiko は localhost に接続不可）
> - Ollama 側で `OLLAMA_ORIGINS` の設定（aoiko の URL を許可）が必要

### 7-C. Tesseract（純ローカル WASM OCR）

LLM を使わず、ブラウザ内で完結する OCR。**精度は限定的**で、人手確認前提。

1. エンジン選択で **「Tesseract」**
2. **「保存」**
3. （任意）`traineddata` の自己ホスト URL を **langPath** に指定（既定は CDN）

> - 画像は端末外に出ません（traineddata の初回 DL のみ CDN）
> - 抽出するのは T+13 登録番号・日付・合計のみ。店名・品目は推定しません
> - 領収書 OCR 専用で、CSV 分類・注文取込には使えません

## 9. 動作確認：初めての仕訳を作ってみる

設定が一通り終わったら、ナビ **「ホーム」** に戻って簡単な仕訳を 1 つ作ってみてください。

例：手元の現金 100,000 円を事業に投入

| 日付 | 摘要 | 借方 | 貸方 | 金額 |
|---|---|---|---|---|
| 今日 | 元入 | 1110 現金 | 3110 元入金 | 100,000 |

詳しい手順は次章 [02. 仕訳の作成](02-journal.md) へ。
