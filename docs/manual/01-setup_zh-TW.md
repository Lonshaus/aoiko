# 01. 初次設定

從啟動 aoiko 到能開始記帳的步驟。

**Language**: [日本語](01-setup.md) | [English](01-setup_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 同意免責、登記商號／年度／消費税方式
> - 登記輔助科目（例如各銀行帳戶分開）和交易對象
> - 設定 OCR/LLM 的 API 金鑰／endpoint（要用到才需要）
>
> **前提**：照[主 README](../../README_zh-TW.md) 的步驟把 aoiko 跑起來、瀏覽器打開了。

## 1. 同意免責事項

第一次啟動會在畫面中央跳**免責事項對話框**。aoiko 輸出的數值與帳簿不保證正確。實際申告請務必跟稅理士或稅務署確認。

看完點**「同意」**進入本體畫面。同意紀錄存在 IndexedDB、之後不會再跳（除非免責文有重大改版）。

> 細節請看 [DISCLAIMER_zh-TW.md](../../DISCLAIMER_zh-TW.md)。

## 2. 切換語言（選填）

點導覽列右邊**「設定」**→ 最上面**「語言」**區塊 → 選 `日本語 / English / 繁體中文` → 按**「儲存」**。

> 預設是日文。本指南雖以日文 UI 為基準描述，繁體中文 UI 標籤都列在旁邊。

## 3. 登記基本資訊

打開設定畫面的**「基本資訊」**區塊。這會出現在決算書・確定申告書 B・`.xtx` 檔案上的事業者資訊。

| 欄位 | 填什麼 | 範例 |
|---|---|---|
| 事業名 / 商號 | 確定申告書商號欄要顯示的名稱 | `青井網路工作室` |
| 發票登錄號碼 | 適格請求書発行事業者登錄號碼 | `T1234567890123` |
| 處理中的年度 | 要記帳的年度 | `2026` |

填完按**「儲存」**。沒有登錄發票的話登錄號碼留空。

## 4. 選擇消費税方式

設定畫面的**「消費税」**區塊選擇申告計算方式（本工具只提供**概算與比較**，實際申告書輸出不在範圍）。

### 4-1. 納税義務

- **課税事業者**：要做消費税申告。下方再選課税方式
- **免税事業者**：不用消費税申告（売上 1,000 萬日圓以下 ＆ 沒登錄發票）

### 4-2. 課税方式（課税事業者）

| 方式 | 適合 |
|---|---|
| 本則課税 | 仕入多、想充分利用仕入税額控除 |
| 簡易課税 | 売上 5,000 萬日圓以下、仕入較少、想要計算簡單 |
| 2 割特例（2023/10〜2026/9）| 配合 invoice 制度開始的 3 年限定減負措施 |
| 3 割特例（令和 9・10）| 売上 5,000 萬日圓以下的新設特例 |

選簡易課税要再選**「簡易課税 事業区分」**第 1〜6 種：

- 第 1 種：批發
- 第 2 種：零售、農林漁業（飲食品）
- 第 3 種：製造・建設・農林漁業（其他）
- 第 4 種：其他（餐飲業等）
- 第 5 種：服務・金融・運輸通訊
- 第 6 種：不動産業

按**「儲存」**。

> 方式怎麼選詳見「消費税」章（執筆中）。猶豫的話先選本則課税、年末看「報表」畫面的**消費税 4 方式比較**用實際數字判斷。

## 5. 登記輔助科目（帳戶／經費細目用）

有多個銀行帳戶、或想把同一個經費科目按交易對象細分時用。

設定畫面的**「輔助科目」**區塊：

1. **「親科目」**下拉選一個科目（例：`1130 普通預金`）
2. **「輔助科目名」**填名稱（例：`三菱UFJ 本店`）
3. 按**「新增」**

常見登記例子：

| 親科目 | 輔助科目名 | 用途 |
|---|---|---|
| 1130 普通預金 | 三菱UFJ 本店 | 分帳戶看餘額 |
| 1130 普通預金 | SBI新生 | 同上 |
| 2120 未払金 | 楽天卡 | 分卡看未払金 |
| 2120 未払金 | au PAY 卡 | 同上 |
| 5150 通信費 | AWS | 經費細分 |
| 5150 通信費 | 手機 | 同上 |

> CSV 匯入時的**已知側科目**可以選這裡登記的輔助科目（例：「楽天カード CSV → 2120 未払金 / 楽天卡」）。

## 6. 登記交易對象

把發票登錄號碼或預設對應科目跟交易對象綁起來，CSV 匯入時可以靠**自動分類規則**從「這個交易對象 → 這個科目」自動套用。

設定畫面的**「交易對象」**區塊：

1. 填**「交易對象名」**
2. 選**「區分」**：法人 / 個人 / 公共 / 海外 / 不明
3. **「發票號碼」**（選填）：`T` + 13 位數字
4. **「預設對應科目」**（選填）：這個交易對象常見的經費科目（例：`AWS → 5150 通信費`）
5. 按**「新增」**

> 交易對象別的集計與傳票列表的篩選也都會參照這裡登記的交易對象。

## 7. 要用 OCR/LLM 才做的設定

只在要用 OCR（收據）・LLM 分類（CSV 自動分類）・訂單取込（貼上 Amazon / 楽天 等網頁）時需要。用不到就跳過。

設定畫面**「LLM 連動」**區塊選**「OCR/LLM 引擎」**：

### 7-A. Google Gemini（預設・雲端）

有免費額度、設定最簡單。

1. 到 [Google AI Studio](https://aistudio.google.com/apikey) 申請 API 金鑰（要 Google 帳號）
2. aoiko 的**「Gemini API 金鑰」**貼進去
3. **「儲存」** → **「連線測試」**確認 `✓ 連線成功`

> **注意**：CSV 行與收據圖片會送到 Google 伺服器。送出前會跳**確認對話框**。資料按 Google 隱私政策處理，免費額度可能被用於訓練。詳見 [PRIVACY_zh-TW.md](../../PRIVACY_zh-TW.md)。

### 7-B. OpenAI 相容 / Ollama 等（可本地）

自己跑 Ollama / LM Studio / llama.cpp / vLLM 在本機或任意伺服器時。

1. 引擎選**「OpenAI 相容 / Ollama 等」**
2. 填**「endpoint（baseURL）」**：例 `http://localhost:11434/v1`
3. 填**「API 金鑰」**（Ollama 等本地通常不需要、留空 OK）
4. 按**「取得模型清單」** → 分別選 OCR 用模型 / 分類用模型
5. **「儲存」** → **「連線測試」**

> - OCR 必須選 **vision 對應模型**（llama3.2-vision / qwen2-vl / minicpm-v 等）
> - 指向 localhost 的話、aoiko 本身也要**本機執行**（`pnpm run preview`）。HTTPS 配信版的 aoiko 連不到 localhost
> - Ollama 那邊要設 `OLLAMA_ORIGINS` 把 aoiko 的 URL 加進白名單

### 7-C. Tesseract（純本地 WASM OCR）

不用 LLM、整個在瀏覽器跑完的 OCR。**精度有限**、要人工確認。

1. 引擎選**「Tesseract」**
2. **「儲存」**
3. （選填）`traineddata` 的自架 URL 填到 **langPath**（預設用 CDN）

> - 影像不會離開本機（只有 `traineddata` 第一次從 CDN 抓）
> - 只抽 T+13 登錄號碼・日期・合計、店名與品項不會推測
> - 收據 OCR 專用、CSV 分類與訂單取込用不到

## 9. 動作確認：建第一張傳票

設定都做完之後，回導覽列**「首頁」**試做一張簡單的傳票。

例：把手上的現金 100,000 円投入事業

| 日期 | 摘要 | 借方 | 貸方 | 金額 |
|---|---|---|---|---|
| 今天 | 元入 | 1110 現金 | 3110 元入金 | 100,000 |

詳細步驟在下一章 [02. 建立傳票](02-journal_zh-TW.md)。
