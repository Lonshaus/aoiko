# 12. 修正申告

申告済年度のスナップショット作成、差分検出、修正仕訳の入れ方、提出手順。

**Language**: **日本語** | [English](12-amended_en.md) | [繁體中文](12-amended_zh-TW.md)

> **⚠ 訂正（2026-06）**：本章の「e-Tax ソフト(WEB 版)」での修正申告手順は誤り。**所得税は WEB 版 非対応**のため、修正申告書は **確定申告書等作成コーナー** または **e-Tax ソフト ダウンロード版（Windows）** で作成・送信すること（[10. `.xtx` 出力](10-xtx-export.md) の訂正注記も参照）。

> **この章を読み終えたら出来ること**
> - 申告済の年度をロックして「申告時点のスナップショット」を保持できる
> - 後から間違いに気付いた時、差分を見て修正仕訳を入れられる
> - 修正申告書の提出に必要な情報を整理できる
>
> **前提**：[06. レポート](06-reports.md) で当該年度をロック済（[§ 8](06-reports.md#8-年度ロック申告済み)）。

## 1. 修正申告とは

確定申告書を提出した **後で**、税額計算や所得計算に誤りがあったと判明した場合：

- **税額を過少に申告していた**（追加納税が必要）→ **修正申告**
- **税額を過大に申告していた**（還付の権利）→ **更正の請求**

両者は提出様式・処理が異なります。aoiko 側の作業は両者ともほぼ同じ（変更を入れて差分を出す）ですが、e-Tax 側の手続が異なります。

> 本ツールは申告書の作成・提出を直接は支援しません。差分検出と「何が変わったか」の整理のみ提供します。

## 2. 申告済スナップショット

[06. レポート § 8](06-reports.md#8-年度ロック申告済み) で **「申告済みとしてロック」** を実行すると、aoiko はその時点の：

- 各勘定科目別の合計（PL・BS 全項目）
- 主要レポート（月別売上、固定資産明細 等）

…を **snapshot** として IndexedDB に保存します。これが申告書提出時に書いた内容のスナップショット。

> 申告書原本（紙 or e-Tax 提出データの控え）は **必ず別途保管**してください。aoiko の snapshot は数値ベースで、申告書本体（個人情報・所得控除等）まではカバーしません。

## 3. 修正のワークフロー

### 3-1. ロック解除

設定変更を加えるためロックを解く必要があります。

1. レポート → 該当年度を選択
2. 🔒 申告済みバッジ横の **「ロック解除」** をクリック
3. 確認後、年度がロック解除状態に

> snapshot は削除されず保持されます（重要）。

### 3-2. 差分の対象を入れる

ロック解除後：

- 仕訳の **追加**（漏れていた経費、申告漏れの売上 等）→ 通常通り [02. 仕訳の作成](02-journal.md)
- 仕訳の **訂正**（誤った金額・科目）→ [02. § 3 訂正仕訳](02-journal.md#3-訂正仕訳修正仕訳--間違いを正す) で打ち消し → 正しい仕訳を新規追加
- 固定資産の **再登録**（耐用年数の修正、廃棄処理 等）→ [08. 減価償却](08-depreciation.md)

> 修正仕訳の `description` は分かりやすく書く（例：「2026 年分修正申告 — 売上漏れ追加（XXX 社）」）。後で差分の理由が分かるように。

### 3-3. 差分の確認

修正仕訳を入れた後、レポート → **概況**ブロックを見ると：

- 現在値（修正後）と snapshot の **差** が表示される
- 各項目の `Before → After` が並ぶ
- 主な変化（PL の所得増減、BS の残高変化）の要約

> 差分が想定どおりか必ず確認。意図しない箇所が変わっていたら関連仕訳を見直す。

### 3-4. e-Tax 側で修正申告書を作成

aoiko 上で数値が固まったら：

1. e-Tax ソフト(WEB 版) にログイン
2. **「修正申告書の作成」** メニュー（正確な名称は当年度の e-Tax UI に依存）
3. **当初申告（青色申告決算書 → 確定申告書）の値**は手動で再入力（または `.xtx` 再出力で読込）
4. **「正しい金額」** を修正後の数値で入力
5. 差分計算は e-Tax 側で自動。延滞税・加算税の概算も表示される
6. 提出（電子）

> aoiko の `.xtx` 再出力（[10. `.xtx` 出力](10-xtx-export.md)）でも修正後の決算書が出るので、それを参考に手動入力する流れも有効。

### 3-5. 提出後の再ロック

修正申告書を提出したら：

1. レポート → 該当年度 → **「申告済みとしてロック」** で再ロック
2. **新しい snapshot で上書き**される（修正後の状態が「申告済み」値に）

> 古い snapshot は失われます。「修正申告前 vs 修正申告後」の比較は事後できないため、申告書原本（提出データ）が結局の真実の記録。

## 4. ありがちなケース

### ケース A：売上の計上漏れ

```
発覚：2026 年分の確定申告提出後、12 月分の売上 100,000 円が漏れていた
```

1. ロック解除
2. 仕訳追加：`借方 1320 未収入金 100,000 / 貸方 4110 売上高 100,000`（日付 2025-12-31 等）
3. 差分確認：所得 +100,000
4. e-Tax で修正申告 → 追加納税
5. 再ロック

### ケース B：経費の二重計上

```
発覚：同じ AWS 費用 5,000 円が CSV 取込と手動仕訳で 2 回入っていた
```

1. ロック解除
2. [03. CSV § 5](03-csv-import.md#5-取込履歴とバッチ訂正) で該当バッチを reverse、または手動の方を訂正仕訳で打消し
3. 差分確認：経費 -5,000、所得 +5,000
4. e-Tax で修正申告 → 追加納税
5. 再ロック

### ケース C：減価償却の耐用年数誤り

```
発覚：4 年で償却していた MacBook が、法的には 4 年で正しいことを再確認。実は経費計上漏れ
```

1. ロック解除
2. 固定資産 → 該当資産を確認 → 取得日・耐用年数・科目を見直し
3. 既存の減価償却仕訳を訂正（reverse → 正しい計算で再生成）
4. 差分確認
5. e-Tax で修正申告
6. 再ロック

### ケース D：所得控除の修正（aoiko 範囲外）

```
発覚：医療費控除を入れ忘れていた
```

aoiko は所得控除を扱わない。e-Tax 側で **「更正の請求」** を直接行う。aoiko 上の操作は不要（事業所得側に変更が無いため）。

## 5. 期限と注意

### 修正申告の提出期限

- **修正申告**（追加納税）：いつでも可。気付いたら早く提出（延滞税が日々増える）
- **更正の請求**（還付）：原則、申告期限から **5 年以内**

### 加算税・延滞税

- 修正申告の場合、**過少申告加算税**（10% or 15%）と **延滞税**（年率変動）が課される可能性
- 税務署からの調査通知前に自発的に修正すれば、加算税が軽減または非課税
- 詳細は税理士・税務署に確認

### 帳簿保存

- 修正申告後の **修正仕訳**は電子帳簿保存法的に 7 年間保存対象
- 修正前後の差分は aoiko 内では「訂正仕訳の履歴」として残るので、これも保存対象
- バックアップ（[11. バックアップと復元](11-backup.md)）でしっかり保管
