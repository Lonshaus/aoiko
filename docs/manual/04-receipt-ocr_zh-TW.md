# 04. 收據 OCR

從紙本收據圖片建立傳票候選。

**Language**: [日本語](04-receipt-ocr.md) | [English](04-receipt-ocr_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 從照片或 PDF 圖片抽出日期・店名・合計・適格請求書登錄番号、然後轉成傳票
> - 看懂 3 個 OCR 引擎（Gemini / OpenAI 相容 / Tesseract）的差異與選擇
> - 看懂雲端送出前確認對話框的意義與「下次起不問」要不要勾
>
> **前提**：[01. 初次設定 § 7](01-setup_zh-TW.md#7-要用-ocrllm-才做的設定) 設好 OCR/LLM 引擎。

## 1. 引擎挑選（速查）

| 引擎 | 精度 | 是否離開本機 | 需要的東西 |
|---|---|---|---|
| **Gemini Vision**（預設）| ◎ | 是 | Gemini API 金鑰 |
| **OpenAI 相容**（Ollama 等）| ◯〜◎ | localhost 否 / 遠端 是 | endpoint ＋ vision 模型 |
| **Tesseract** | △ | 否（只首次 DL traineddata）| 設定中選擇即可 |

> 詳細設定步驟看 [01. § 7](01-setup_zh-TW.md#7-要用-ocrllm-才做的設定)。

## 2. 取込步驟

導覽列「**收據**」進入 `Receipt` 畫面。

### 2-1. 選圖片

點「**1. 選擇圖像（亦可用相機拍攝）**」的檔案輸入：

- PC：檔案對話框選 JPG / PNG / WebP / HEIC
- 手機（裝 PWA）：相機選項會出現、可現場拍

選好後下方會顯示預覽。

> **拍攝小技巧**：
> - 平面、避免陰影
> - 整張收據在框內
> - 避開螢光燈反光
> - 1〜2 MP 解析度就夠。再高反而對 LLM rate limit 不利

### 2-2. 解析

按「**解析**」送到選中的引擎開始解析。

#### 雲端引擎（Gemini / 遠端 OpenAI 相容）：送出前確認對話框

因為會離開本機，按下去馬上跳 **CloudSendConfirmDialog**：

> 要送出圖片到 〇〇 嗎？
>
> ☐ 下次起不問

- 確認後按「**送出**」執行
- 「**取消**」中止
- 勾「下次起不問」後送出、之後同一引擎不再跳對話框

> **勾之前請三思**：這會存到 IndexedDB `skipExternalSendConfirm: true`（全引擎共通旗標）。沒有獨立 UI 開關可解除。誤勾的話請從「設定 → 資料管理」重置該 key（或手動編輯 IndexedDB）。

#### Tesseract：不跳對話框

WASM 在本機處理、不需要確認。但解析時 `tesseract.js` 首次會從 CDN 抓 traineddata（數 MB）。

### 2-3. 抽取結果確認・修正

解析完成後「**2. 抽取結果（可編輯）**」會展開：

| 欄位 | 內容 |
|---|---|
| 交易日 | 收據日期、`YYYY-MM-DD` |
| 店名 / 交易對象 | 店名（可編輯）|
| 合計金額（円）| 含稅合計（可編輯，整數）|
| 發票登錄號碼 | 認得到 T+13 位時才顯示（參考用）|
| 內訳 | 有品項時可展開（參考用、不會反映到傳票）|

> **vision LLM 路 vs Tesseract 路的差別**：
> - vision LLM 店名・合計都能高精度抽出。連品項內訳也撈
> - Tesseract **只抽日期・合計・T+13 登錄號碼** 這 3 種以確定性規則拿。**店名與品項不推測、保留空白**。OCR 全文內部留著但不自動寫進傳票摘要

#### Tesseract 時的警告 banner

Tesseract 路解析後、抽取結果 header 下方會跳黃色注意 banner：

> 這是純本地 OCR（Tesseract）的結果，精度有限。請務必確認並修正合計金額、日期與店名。

### 2-4. 選對方科目・支付來源

抽取結果下方有 2 個下拉：

| 下拉 | 預設 | 意思 |
|---|---|---|
| **對方科目（經費）**| `5910 雑費` | 這張收據要計上的經費科目 |
| **支付來源** | `1110 現金` | 用什麼付的（現金・普通預金・未払金 等）|

如果是事業用、無家事分攤的單純經費，改成對應科目（例：餐飲 → `5170 接待交際費`、文具 → `5200 消耗品費`）。

刷卡的話**支付來源**改成 `2120 未払金`（有用輔助科目分卡的話、選該卡）。

### 2-5. 建立傳票

按「**建立傳票**」確定。借方＝經費／貸方＝支付來源 的 2 行傳票建立、首頁最近一覽會出現。

> 抽取結果有**發票登錄號碼**的話，該傳票行的 `invoiceCompliant` flag 會存成 `true`（消費税本則課税時可作為仕入税額控除對象）。

## 3. 各引擎實用 tips

### Gemini Vision

- 免費額度月幾百張不成問題、收據圖片 token 消耗少
- 拍歪一點、有部分陰影也撐得住
- 1〜2 MP 解析度就夠、再高反而拖速度・成本

### OpenAI 相容（Ollama / LM Studio 等）

- **必須選 vision 模型**（純文字模型遇到圖片輸入會掛）
- 推薦：`llama3.2-vision`、`ministral-3`、`moondream` 系
- Ollama 那邊：`OLLAMA_ORIGINS` 一定要把 aoiko 的 URL（例 `http://localhost:31527`）加進來
- aoiko 自己也要本機起（`npm run preview`）。HTTPS 配信版連不到 localhost

### Tesseract

- **vision LLM 不能用／不想用**時的選項
- 預設店名・品項抽不到、心理上接受
- T+13 登錄號碼相當穩（regex 抽出）
- 日期・合計撈得到算運氣好。務必人工確認・修正
- 想完全離線、設定中把 `traineddata` 換成自架 URL

## 4. 抽取不出來時的對策

| 症狀 | 對策 |
|---|---|
| 合計金額對不上 | 重拍 or 手動修正。LLM 路通常很少，Tesseract 路常發生、本來就要人工 |
| 日期空 | 「YYYY/MM/DD」「令和○年」兩種格式都試過了。直接手動輸入也行 |
| 店名亂碼 | LLM：拍攝品質・解析度提升。Tesseract：原本就不抽店名、放棄手動補 |
| 發票號碼沒認出來 | 號碼印字部分要亮、重拍。LLM 系 1〜2 mm 小字也能抓 |
| 連線錯誤 | 設定 →「連線測試」確認 API 金鑰・endpoint |

## 5. 隱私注意

- 收據有第三方個資（顧客名・住址）時，送雲端引擎前要再確認
- 高機密的收據（個人醫療費・特殊事業的取引先 等）用 Tesseract 或 Ollama on localhost 比較安心
