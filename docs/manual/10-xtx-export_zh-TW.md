# 10. `.xtx` 輸出

產生 e-Tax 格式的 `.xtx` 檔案，並組み込み（載入）到 e-Tax 軟體（下載版）。

**Language**: [日本語](10-xtx-export.md) | [English](10-xtx-export_en.md) | **繁體中文**

> **讀完本章你能做到**
> - 將確定申告書（KOA020）+ 對應的決算書（青色申告是青色申告決算書一般用 KOA210，白色申告是収支内訳書一般用 KOA110）輸出成一個 `.xtx`
> - 有輸入不動產所得時，不動產所得用的決算書（青色：KOA220、白色：KOA130）也會一起附在同一份 `.xtx`（參見[14. 所得控除・稅額扣抵](14-income-deductions_zh-TW.md)）
> - 把輸出的 `.xtx` 組み込み到 e-Tax 軟體（下載版）
> - 理解 aoiko 填寫的範圍（事業部分）與你需在 e-Tax 補完的範圍（所得控除・稅額・白色申告的専従者控除）
>
> **前提**：[01. 初次設定](01-setup_zh-TW.md) 的基本資訊與申報者資訊、[09. 前期繰越](09-carryover_zh-TW.md) 的期首振替傳票、[08. 減價償卻](08-depreciation_zh-TW.md) 的年末減價償卻已完成，且當年度傳票已確定。

## 1. `.xtx` 是什麼

`.xtx` 是 **e-Tax 軟體（下載版）** 以「組み込み」載入的 XML 申報資料檔。aoiko 依 **國稅廳官方 W3C XSD**（e-tax19「XML スキーマ」來源）將以下併載於一個 `.xtx`：

- **KOA020-023**：確定申告書 第一表
- **KOA210-011**：青色申告決算書（一般用），或白色申告用的 **KOA110-012**：収支内訳書（一般用）——由**設定 → 申報者資訊 → 確定申告方式**決定併載哪一張
- **KOA220-008**：青色申告決算書（不動產所得用），或白色申告用的 **KOA130-009**：収支内訳書（不動產所得用）——只有在**設定 → 使用不動產所得**開啟、且[14. 所得控除・稅額扣抵](14-income-deductions_zh-TW.md)有輸入不動產所得時，才會跟上面的事業用決算書一起附上
- **TEA060**：申告書等送信票（兼送付書）
- **手續代碼**：`RKO0010`（所得稅及復興特別所得稅申告）

aoiko 的 `.xtx` 通過 CI 的 **xmllint 官方 XSD 驗證**，並已在 e-Tax 軟體（下載版） **實機組み込み驗證通過**。

> **關於不動產所得的呆帳沖銷・呆帳準備金提列**：青色申告決算書（不動產所得用 KOA220）沒有這兩個科目的專屬欄位，因此會輸出到表單本身設計的「追加科目」欄位（上限5筆）。收支內訳書（不動產所得用 KOA130）有「貸倒金」的專屬欄位，所以會直接輸出到那裡；但因為白色申告本來就沒有準備金提列的制度，「貸倒引当金繰入額（不動産）」不會輸出到 KOA130（詳見 [14. 所得控除・稅額扣抵](14-income-deductions_zh-TW.md)）。

> **e-Tax 軟體（WEB 版） 不支援所得稅申告（RKO0010） 的組み込み**，請務必使用 **下載版**（需逐稅目安裝模組）。作成コーナー 不支援 `.xtx` 的組み込み。

## 2. aoiko 的 `.xtx` 會填什麼・不填什麼

aoiko 處理的是 **事業損益**。`.xtx` 會帶入決算書、確定申告書的 **事業部分**，以及提出所必須的 **申報者資訊**。

### 會輸出的內容

| 帳票 | 值的來源 |
|---|---|
| 申報者資訊（稅務署・利用者識別號碼・姓名・地址）| Settings 的申報者資訊 |
| 確定申告書 第一表：營業等收入金額（ア）| 報表的 PL（売上）|
| 確定申告書 第一表：事業所得（①）| PL（經費控除後的額。青色申告會再扣除青色申告特別控除）|

**青色申告的情況**（決算書：青色申告決算書一般用 KOA210）

| 帳票 | 值的來源 |
|---|---|
| 確定申告書 第一表：青色申告特別控除額 | PL ＋ 青色控除區分 |
| 決算書 第 1 頁 損益計算書（含控除前/控除額/控除後）| 報表的 PL ＋ 青色控除區分 |
| 決算書 第 2 頁 月別売上（收入）金額・仕入金額 | 報表的月別（仕入僅含売上原価・仕入）|
| 決算書 第 4 頁 貸借対照表 | 報表的 BS |

**白色申告的情況**（決算書：収支内訳書一般用 KOA110）

| 帳票 | 值的來源 |
|---|---|
| 収支内訳書 第 1 頁 收入金額・經費各科目・専従者控除前的所得金額 | 報表的 PL |

> 白色申告沒有貸借対照表・月別売上的對映欄位（因為収支内訳書本身就沒有這些項目）。専従者給与・貸倒引当金繰入額在収支内訳書沒有對應欄位，所得計算也會排除它們（詳見下方「不會輸出的內容」）。

### 不會輸出的內容（在 e-Tax 補完）

| 內容 | 補完方式 |
|---|---|
| **各項所得控除**（基礎控除、配偶者控除、社会保険料控除、医療費控除 等）| 在 e-Tax 輸入 |
| **合計所得・課稅所得・稅額計算**（所得稅、復興特別所得稅）| e-Tax 自動計算 |
| **附件**（医療費明細、各項控除證明 等）| 在 e-Tax 附加 |
| **消費稅申告書**（跨多事業區分的簡易課税・3割特例）| 另在 e-Tax 製作（aoiko 僅提供概算；本則課税・2割特例・單一事業區分的簡易課税支援 `.xtx` 輸出：[§6](#6-消費税本則課税2割特例簡易課税的-xtx-輸出)、[07. 消費稅](07-consumption-tax_zh-TW.md)）|
| **白色申告的専従者控除**（配偶者86萬圓・其他親屬每人50萬圓的定額控除）與控除後所得 | 在 e-Tax 輸入（需要續柄資訊，aoiko 不算定）|

> 這是會計軟體共通的分工：記帳・決算書・事業所得由軟體產出，個人所得控除與稅額在確定申告階段由本人於 e-Tax 補完。

## 3. 輸出步驟

### 3-1. 輸出前檢查

1. **設定 → 申報者資訊** 已填 稅務署・利用者識別號碼・姓名・地址（未填則無法輸出 `.xtx`）
2. **設定 → 確定申告方式** 與實際申告方式一致（青色申告／白色申告）
3. 青色申告的話，**設定 → 青色申告特別控除** 區分（65萬/55萬/10萬）正確
4. 當年度 **傳票全部確定**
5. **期首振替傳票**（[09. 前期繰越](09-carryover_zh-TW.md)）已以當年度 `2026-01-01` 建立
6. **減價償卻傳票** 已於年末產生（[08. § 3](08-depreciation_zh-TW.md#3-年末產生減價償卻傳票)）
7. 青色申告的話，報表 → BS 的 **「資產合計」與「負債・純資產合計」一致**（無不一致警告）。白色申告的収支内訳書沒有貸借対照表，不用確認這項

### 3-2. 檔名

aoiko 產生的檔名為 `aoiko-{年度}.xtx`，存到瀏覽器的「下載」資料夾。

> aoiko 的 `.gitignore` 已含 `aoiko-*.xtx`，不會誤 commit 進 repo。

> **關於令和8年分**：令和8年分（2026 年分）的 e-Tax 所得稅模組要到申告期（2027 年）才釋出。可用開發版的「テスト: 令和7年分で書き出し」按鈕，對目前提供的令和7年分模組做動作驗證（封包結構與年分無關）。

## 4. 在 e-Tax 軟體（下載版） 組み込み

**重要**：aoiko 的 `.xtx` 已在 CI 與實機驗證，但 **申告內容的最終確認仍是使用者責任**。

1. 啟動下載版，**追加安裝所得稅的年分模組**（若尚未安裝）
2. **選擇利用者** → 「**申告・申請等**」
3. 「**組み込み**」 → 選 aoiko 輸出的 `.xtx`
4. 確認 3 張帳票（確定申告書・決算書〔青色申告決算書 或 収支内訳書〕・送信票）都變成「組み込み」狀態
5. 用 「**帳票編集**」 核對數值：
   - 青色申告：決算書（一般用）的損益・月別・貸借対照表 與 aoiko 報表一致
   - 白色申告：収支内訳書（一般用）的收入・經費各科目・専従者控除前的所得金額 與 aoiko 報表一致
   - 確定申告書 第一表 的 營業等收入・事業所得（僅青色申告有・青色申告特別控除額）・姓名・地址・稅務署 正確
6. 在 e-Tax 輸入其餘項目（各項所得控除、稅額計算、白色申告的専従者控除 等）
7. **組み込み 時若出錯**：可能是本工具輸出有問題，請用 GitHub Issue 回報。

## 5. 內部規格（參考）

### XSD 來源

- 發布處：國稅廳 e-Tax「仕様公開資料」一覽
- 原 CAB：`e-tax19.CAB`「XML スキーマ」
- aoiko 同梱：`docs/xtx-spec/{shotoku,general}/` 之下
  - `KOA020-023.xsd`（確定申告書）・`KOA210-011.xsd`（青色申告決算書 一般用）・`KOA110-012.xsd`（収支内訳書 一般用）
  - `KOA220-008.xsd`（青色申告決算書 不動產所得用）・`KOA130-009.xsd`（収支内訳書 不動產所得用）
  - `General.xsd`、`ITdefinition.xsd`、`ITreference.xsd`、`zeimusho.xsd`、`zeimoku.xsd`

### 文件模型

2 段式 ID/IDREF：

- **封包**：`<DATA>`（5 個命名空間）→ 手續元素 `RKO0010` → `CATALOG`（RDF manifest）＋ `CONTENTS`
- **定義側**（IT 部）：`<IT VR="1.5" id="IT">` 內以 ID 列舉申報者資訊・年分 等
- **參照側**（帳票個別部分）：每張樣式為單一樣式元素，內含頁子元素，leaf 以 IDREF 指向定義側
- **送信票**：`SOFUSHO`（TEA060、kyotsu 命名空間）

aoiko 的 `.xtx` 產生是依 XSD 自動生成的 JSON schema 動態組裝。

## 6. 消費税（本則課税・2割特例・簡易課税）的 `.xtx` 輸出

跟上述所得税 `.xtx`（手續 `RKO0010`）是**不同檔案・不同手續**。消費税課税期間（暦年）確定後，要跟所得税分開組み込み。

### 支援範圍

- **手續 `RSH0010`**（消費税及び地方消費税申告），輸出以下其中一種：
  - **本則課税**：消費税及び地方消費税の申告書（一般用）＋付表1-3＋付表2-3。**假設課税売上割合100%**（沒有非課税銷售・輸出免税等）——付表2-3 只使用「課税売上高5億円以下・課税売上割合95%以上」的全額控除區分，個別対応方式・一括比例配分方式不支援
  - **2割特例**：消費税及び地方消費税の申告書（簡易課税用）＋付表6（税額控除に係る経過措置＝2割特例）。2割特例即使沒有正式選擇簡易課税，也是用這個様式結構申告（依國税庁2割特例専用手引き）
  - **簡易課税（僅限單一事業區分）**：消費税及び地方消費税の申告書（簡易課税用）＋付表4-3＋付表5-3
- **目前只支援本則課税・2割特例・簡易課税（單一事業區分）**。跨多個事業區分的簡易課税（75%規則等）・3割特例的申告書輸出尚未支援（見[07. 消費税](07-consumption-tax_zh-TW.md)）
- **已知的簡化**：売上対価的返還等（退貨・折讓）因為 aoiko 的集計已經淨額計入課税標準額，對應付表欄不會單獨列出這筆明細。**最終税額本身計算是正確的**

### 輸出步驟

1. **設定 → 消費税** 設為「課税事業者」（要輸出簡易課税的話，再設「簡易課税」＋事業區分）
2. **設定 → 申告者情報** 填好税務署・利用者識別番号・氏名・住所
3. 報表頁面 → 「**消費税**」區塊，確認要輸出的方式有出現在列表
4. 點「**下載本則課税的.xtx**」・「**下載2割特例的.xtx**」或「**下載簡易課税的.xtx**」按鈕（檔名 `aoiko-shohi-{年度}.xtx`）

### 在 e-Tax 軟體（下載版） 組み込み

跟所得税分開，先安裝**消費税的年分模組**再組み込み。步驟跟[§4](#4-在-e-tax-軟體下載版-組み込み)相同（「組み込み」→選檔→帳票編集確認數字）。

## 7. 常見錯誤

| 錯誤（e-Tax 端）| 推測原因 |
|---|---|
| `SC00X010 このファイルを組み込むことはできません` | 申報者資訊未填，或投入錯誤年分模組 |
| 檔案格式不符 | 投入了非 `.xtx` 檔 |
| 樣式不一致 | aoiko 同梱 XSD 與當年度版偏離（[docs/xtx-spec/README.md](../../docs/xtx-spec/README.md) 確認是否最新）|
| 數值位數・格式不符 | 金額為負或非整數等，請檢查傳票 |
