# 10. `.xtx` 輸出

產生 e-Tax 格式的 `.xtx` 檔案，並用 e-Tax 軟體（下載版）做匯入驗證。

**Language**: [日本語](10-xtx-export.md) | [English](10-xtx-export_en.md) | **繁體中文**

> **⚠ 重要訂正（2026-06 實機調查）**
> 本章原本以「用 e-Tax 軟體(WEB 版) 匯入」為前提撰寫，但實機調查證實 **WEB 版不支援所得税確定申告（手續 RKO0010）的匯入**（官方「利用可能手続一覧」確認）。

> - **Mac** 沒有任何匯入 client → 實務上請把 aoiko 報表的數字**手動填進作成コーナー**（數字已驗證正確）。
> 以下文中的「WEB 版」步驟請一律改讀為「下載版（Windows）」。`.xtx` 仍為暫行實作，實際申告請依免責聲明。

> **讀完這章你能做到**
> - 把確定申告書（KOA020）+ 青色申告決算書一般用（KOA210）輸出成 1 個 `.xtx`
> - 用 e-Tax 軟體(WEB 版) 匯入驗證輸出的 `.xtx`
> - 知道符合官方 XSD 的意義跟 aoiko 的責任範圍
>
> **前提**：[01. 初次設定](01-setup_zh-TW.md) 基本資訊、[09. 前期繰越](09-carryover_zh-TW.md) 期首振替、[08. 減價償卻](08-depreciation_zh-TW.md) 年末償卻 都完成、當年度傳票確定。

## 1. `.xtx` 是什麼

`.xtx` 是 **e-Tax 軟體(WEB 版)** 接受的 XML 格式申告資料檔。aoiko 按 **国税庁官方 W3C XSD**（e-tax19 「XML スキーマ」來源）把以下這些併載到 1 個 `.xtx` 裡：

- **KOA020-023**：確定申告書第一表〜第四表（令和 8 年分）
- **KOA210-011**：青色申告決算書（一般用）
- **手続コード**：`RKO0010`（所得税及び復興特別所得税申告）

aoiko 的 `.xtx` 結構 CI 每次都跑 **xmllint 對官方 XSD 的 validation**、結構上符合官方規格。

## 2. aoiko 的 `.xtx` 輸出哪些・不輸出哪些

### 輸出內容

| 帳票 | 值的來源 |
|---|---|
| 確定申告書第一表（年分、商號）| Settings 基本資訊 |
| 青色申告決算書第 1 頁 損益計算書 | 報表的 PL |
| 青色申告決算書第 2 頁 月別売上（收入）金額 | 報表的月別銷貨 |
| 青色申告決算書第 3 頁 減価償却費計算 | 固定資產＋當年度償卻 |
| 青色申告決算書第 4 頁 資產負債表 | 報表的 BS |

### 不輸出（要人工 or e-Tax 側補）

| 內容 | 補的方式 |
|---|---|
| 確定申告書本體**個人資訊**（姓名、住址、生年月日、個人番号）| e-Tax 側輸入 |
| **所得控除**（基礎控除、配偶者控除、社會保險料控除、醫療費控除 等）| e-Tax 側輸入 |
| **税額計算**（所得税、復興特別所得税）| e-Tax 側自動計算 |
| **添附資料**（醫療費明細、寄附金控除證明 等）| e-Tax 側添附 |
| **消費税申告書**（付表 2-3 等）| 另在 e-Tax 側做（aoiko 只概算、[07. 消費税](07-consumption-tax_zh-TW.md)）|

> aoiko 的 `.xtx` 是**事業所得部分（決算書）+ 年分・商號** 填好的骨架檔。剩下的在 e-Tax 側補的前提。

## 3. 輸出手順

> 從 aoiko 報表畫面或 `.xtx` 輸出 UI 選單執行（實裝位置依版本不同）。

### 3-1. 輸出前確認

1. 當年度**傳票全部確定**
2. **期首振替傳票**（[09. 前期繰越](09-carryover_zh-TW.md)）在當年度 `2026-01-01` 已建立（初年度的話手動建）
3. **減價償卻傳票**年末已生成（[08. § 3](08-depreciation_zh-TW.md#3-年末產生減價償卻傳票)）
4. 報表 → BS **「資產合計」與「負債・淨資產合計」一致**（沒不一致警告）
5. 設定 → 基本資訊的 **商號、處理中的年度**正確

### 3-2. 檔名

aoiko 產生的檔名是 `aoiko-{年度}.xtx` 等格式（依實裝）。存在瀏覽器「下載」資料夾。

> aoiko 的 `.gitignore` 包含 `aoiko-*.xtx`、不會誤 commit 到 repo。

## 4. e-Tax 軟體(WEB 版) 匯入驗證

**重要**：aoiko 的 `.xtx` CI 過官方 XSD validation、但**實機匯入驗證是利用者責任**。提交前務必：

1. [e-Tax 首頁](https://www.e-tax.nta.go.jp/) → 登入 e-Tax 軟體(WEB 版)
2. **「作成」 → 「ファイルから読込」**或同等選單
3. 選 aoiko 輸出的 `.xtx` 匯入
4. 匯入後在畫面確認：
   - 決算書（一般用）的數值跟 aoiko 報表一致
   - 月別銷貨各月一致
   - 減価償却各資產行一致
   - 資產負債表各項目一致
5. 不足的項目（個人資訊、所得控除、税額計算 等）在 e-Tax 側輸入
6. **e-Tax 側出錯**：本工具的輸出可能有問題。請到 GitHub Issue 回報。

> **驗證為什麼重要**：CI 過 xmllint ≠ e-Tax 側能接收（schema 過但 e-Tax 側業務 logic 弱會被擋的可能性另算）。儘早實機驗證。

## 5. 內部規格（參考）

### XSD 來源

- 配布源：国税庁 e-Tax「仕様公開資料」一覧
- 元 CAB：`e-tax19.CAB`「XML スキーマ」
- aoiko 同梱：`docs/xtx-spec/{shotoku,general}/` 下 7 個檔
  - `KOA020-023.xsd`（確定申告書）
  - `KOA210-011.xsd`（青色申告決算書 一般用）
  - `General.xsd`、`ITdefinition.xsd`、`ITreference.xsd`、`zeimusho.xsd`、`zeimoku.xsd`

### 文件模型

2 段式 ID/IDREF：

- **定義側**（IT 部）：`<IT VR="1.0" id="IT">` 內值要素附 ID 屬性列舉
- **參照側**（帳票個別部分）：各様式的 leaf 用 IDREF 指定義側的空要素

這是 e-tax 標準模型。aoiko 的 `.xtx` 產生腳本依 XSD 自動產生的 JSON schema 動態組裝。

## 6. 常見錯誤

| 錯誤（e-Tax 側）| 推測原因 |
|---|---|
| 檔案格式不對 | 非 `.xtx`、或別年度版投入 |
| 様式不一致 | aoiko 同梱 XSD 跟當年度版有差（[docs/xtx-spec/README.md](../../docs/xtx-spec/README.md) 確認是不是最新）|
| 必須項目不足 | 商號未設等。確認 Settings 基本資訊 |
| 數值的位數・格式不對 | 金額是負值、或非整數輸入 等。檢查傳票 |
