# 15. 請款單・報價單的發行

在「請款單」畫面建立並發行給交易對象的請款單・報價單。

**Language**: [日本語](15-invoices.md) | [English](15-invoices_en.md) | **繁體中文**

> **讀完這章你能做到**
> - 建立並發行請款單・報價單
> - 了解發行時自動產生仕訳・應收帳款的機制
> - 訂正已發行文件的方法（打消し仕訳）
> - 一鍵把報價單轉為請款單
>
> **前提**：[01. 初次設定](01-setup_zh-TW.md) 做完。交易對象（客戶）要先在設定 →「交易對象」登錄好。

## 1. 請款單與報價單的差異

| | 請款單 | 報價單 |
|---|---|---|
| 編號系列 | 獨立（例：INV-2026-0001） | 獨立（例：QUO-2026-0001） |
| 發行時的會計處理 | 自動產生仕訳（借方應收帳款・貸方銷貨收入）＋應收帳款（ArApEntry） | 無（成立前的提案） |
| 訂正方式 | 打消し仕訳（跟 [12. 修正申告](12-amended_zh-TW.md) 同樣的思路） | 發行後不能改內容，只能取消 |

報價單終究只是「還沒成立的提案」，發行也完全不影響帳簿。實際成交後，從報價單一鍵轉換成請款單（§ 4）。

## 2. 建立請款單・報價單

1. 導覽選單 「**請款單**」 選擇「請款單」或「報價單」分頁
2. 點 「**新增**」
3. 輸入對象（交易對象）・交易年月日・（僅請款單）付款期限
4. 明細行輸入品名・數量・單價（未稅）・稅率。「**＋ 新增明細**」 可以增加行數
5. 消費稅額按稅率分組只計算一次（遵循發票制度「每個稅率只做一次進位」的原則，不是逐行明細計算）
6. 確認內容後點 「**儲存草稿**」（之後繼續編輯）或 「**發行**」

> **想在請款單上顯示對象地址**：先在設定 →「交易對象」幫該交易對象登錄送付地址（選填欄位）。

## 3. 發行後會發生什麼

**發行請款單：**

- 確定編號（設定裡指定的前綴 + 年 + 連號，例：`INV-2026-0001`）
- 自動產生仕訳：借方 應收帳款（含稅合計）・貸方 銷貨收入（依稅率分組、含稅金額）
- 產生應收帳款（ArApEntry），反映到 [06. 報表 § 10-2](06-reports_zh-TW.md#10-2-應收帳款應付帳款與現金流預測) 的預計入金
- 內容鎖定（無法退回草稿）

**發行報價單：**

- 只確定編號，不產生任何仕訳・應收帳款

## 4. 報價單 → 請款單的轉換

從報價單列表點 「**轉為請款單**」，會複製明細建立一張新的請款單草稿（交易年月日預設今天，編號尚未確定）。確認・調整內容後再發行。

## 5. 列印送出

已發行的文件在列表點 「**列印**」 會開啟瀏覽器的列印對話框。要存成 PDF，在列印對話框的輸出目的地選「另存為 PDF」。

列印版面會自動包含適格請求書等保存方式要求的記載事項：

- 發行者名稱・登錄號碼（設定 →「基本資訊」的事業者資訊；免稅事業者（未登錄號碼）不會顯示登錄號碼欄）
- 交易年月日
- 交易內容（明細）
- 依稅率分組的合計金額及適用稅率
- 依稅率分組的消費稅額
- 收受文件的事業者名稱（對象）

> aoiko 沒有 PDF 產生或送出（例如寄信）功能。用瀏覽器的列印功能做出 PDF 後，請自行用你的方式送出（email 附件、郵寄等）。

## 6. 訂正・取消已發行的請款單

已發行請款單的內容不能直接編輯（確定仕訳不可竄改原則，跟 [12. 修正申告](12-amended_zh-TW.md) 同樣的思路）。

1. 請款單列表點 「**取消**」
2. 產生打消し仕訳（原仕訳借貸互換），原仕訳保留、標記為已取消
3. 應收帳款（ArApEntry）也會一併刪除
4. 用正確內容重新建立並發行新的請款單

> **有入金記錄的請款單不能取消。** 會破壞消込整合性，請先取消入金記錄（[06. 報表 § 10-2](06-reports_zh-TW.md#10-2-應收帳款應付帳款與現金流預測)）再取消請款單。
>
> 報價單沒有仕訳，內容錯了直接取消重做即可。
