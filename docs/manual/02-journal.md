# 02. 仕訳の作成

**Language**: **日本語** | [English](02-journal_en.md) | [繁體中文](02-journal_zh-TW.md)

手動での仕訳作成、仕訳一覧での閲覧・検索、訂正仕訳（修正仕訳）。

> **この章を読み終えたら出来ること**
> - 借方／貸方の複数行仕訳を手動で入力できる
> - 仕訳一覧を年/月/摘要/金額/取引先で絞り込める
> - 確定済みの仕訳を訂正仕訳で打ち消せる
>
> **前提**：[01. 初回設定](01-setup.md) を済ませてあること（屋号・年度・必要なら補助科目）。

## 1. 手動で仕訳を作る — ホーム画面

ナビ **「ホーム」** の下半分が **新規仕訳フォーム**（`JournalEntryForm`）。

### 1-1. 基本の流れ

1. **日付**：取引日を入力（既定は今日）
2. **摘要**：何の取引か（例：`電気代`、`セミナー受講料`）
3. **借方**を 1 行以上入力：
   - **科目**：ドロップダウンから（例：`5130 水道光熱費`）
   - **補助科目**（任意）：登録済みなら（例：`AWS`）
   - **金額**：半角整数で
   - **税率**：`10%` / `8%` / `0%`（非課税）から
   - **内税**：チェックで税込、外すと税抜
4. **貸方**を 1 行以上入力（同じ要領）
5. **「＋ 借方を追加」** / **「＋ 貸方を追加」** で行追加（複合仕訳）
6. 借方合計と貸方合計が一致すると **「✓ 一致」** の緑バッジ。差額があれば **「差額 ¥xxx」** の警告
7. **「仕訳を追加」** をクリック → 即座にホームの「直近の仕訳」一覧に出る

### 1-2. 不動産所得の仕訳を作る（設定でオプトインした場合のみ）

**「設定」→ 不動産所得を使う** を有効にすると、フォーム上部に **「事業」／「不動産」** の切替が出ます。「不動産」を選ぶと科目のドロップダウンが不動産用科目（末尾「（不動産）」）だけに絞り込まれます。切替は入力中の内容を記憶したまま次の仕訳にも引き継がれるため、同種の取引をまとめて入力する際に毎回選び直す必要はありません。

> 有効にしていない間はこの切替自体が表示されず、既存の画面と完全に同じです。

### 1-3. 家事按分を使う

自宅兼事務所の経費を「事業使用分 ＋ 事業主貸」に自動で分けたい場合。

1. 該当行の **「家事按分」** にチェック
2. 比率を入力（例：`0.30` = 事業使用 30%、残り 70% は自動的に事業主貸へ）
3. 仕訳追加時に 2 行に自動展開（事業経費 + 1610 事業主貸）

> 既定比率は **「設定」→ 家事按分既定比率** で登録できます（行ごとに上書き可）。

### 1-4. クリアする

入力途中で全部やり直したい時は **「クリア」** ボタン。

### 1-5. 品目・数量を記録する（簡易在庫管理）

**仕入（5020）・売上高（4110）** の行を選ぶと、**品目**（ドロップダウン）と **数量** の入力欄が現れます。品目マスタは **「設定」** の商品主檔で登録・削除できます。記録しておくと [06. レポート](06-reports.md) の損益計算書に、法定デフォルトの最終仕入原価法による**期末棚卸高の試算額**が表示されます（自動記帳はされません、参考値として提示するのみ）。

> 届出により最終仕入原価法以外の評価方法を選んでいる場合は、**「設定」→ 評価方法が最終仕入原価法ではない** にチェックすると、この自動試算が無効になり従来通り手動で期末棚卸高を仕訳する運用に戻ります。

### 1-6. 部門タグを付ける

摘要の右の **「部門タグ」** 欄に自由記述でタグを付けられます（例：`東京店`、`ネット販売`）。複数の事業所・拠点・チャネル等を軽く分けて管理したい場合に使います。一度入力したタグは入力補完（オートコンプリート）候補に出ます。

> 部門タグは集計・絞込専用の表示ラベルです。税額計算・`.xtx` 出力には一切影響しません。[06. レポート](06-reports.md) の科目内訳で部門別に絞り込めます。

### 1-7. 証憑写真を添付する

摘要の下の **「証憑写真」** 欄でレシート・領収書の写真を選ぶと、確認ダイアログ（写真プレビュー付き）が出ます。内容を確認して **「添付する」** を押すと、仕訳の確定と同時に写真も保存されます。複数枚添付する場合は選択を繰り返してください（送信前ならサムネイルの **×** で取り消せます）。

> **確定後は差し替え・削除ができません**（電子帳簿保存法の真実性確保の要件）。添付を間違えた場合は訂正仕訳で対応し、元の写真は履歴として残ります。
>
> 確認ダイアログは **「設定」→ 証憑写真の添付前確認をスキップする** で省略できます（後からいつでも再度チェックを外して復活できます）。

### 1-8. 外貨建取引を記帳する

aoiko に多通貨・為替レート取得の専用機能は無い。実務上ほとんどの外貨建取引はクレジットカード明細・銀行送金票等、**発生時点で日本円換算済みの金額が確定している**ため、その日本円金額をそのまま通常の仕訳として入力すればよい（摘要に `USD 100` のように外貨額を書いておくと後で見返しやすい）。

未決済の外貨建売掛金・買掛金がある場合や、海外での現金払いで日本円換算額を自分で計算する必要がある場合も、その時点で妥当と判断した日本円金額で通常通り記帳する。決済時に実際の金額が確定したら、差額を「為替差損」「為替差益」（**「設定」→ 勘定科目** で新規作成）として追加の仕訳で調整する。

> **なぜ専用機能が無いか**：freee・マネーフォワード共に個人事業主向けプランの標準機能では自動レート取得・自動為替差損益計算を提供していない（freeeは外貨建取引管理アプリという別売アプリでのみ対応）。実務上は「発生時点の金額をそのまま記帳し、決済時の差額を為替差損益で調整する」運用で十分なため、aoikoも同じ方針を採る。

### 1-9. 従業員給与・源泉徴収を記帳する

aoiko に給与計算・源泉徴収税額の自動計算機能は無い。専従者（家族従業員）以外の一般従業員を雇っている場合、源泉徴収税額は国税庁公表の「[給与所得の源泉徴収税額表](https://www.nta.go.jp/publication/pamph/gensen/zeigakuhyo2026/01.htm)」で自分で算出し、通常の仕訳として記帳する。

1. **借方**：`5230 給料賃金` に給与総額（社会保険料控除前）
2. **貸方**：`1130 普通預金` 等に実際の振込額（手取り）
3. **貸方**：`2xxx 預り金`（無ければ「設定」→ 勘定科目 で新規作成）に源泉徴収税額＋社会保険料の本人負担分

> **なぜ専用機能が無いか**：freeeの個人事業主向け「確定申告」でも、源泉徴収税額の自動計算・按月試算は提供していない（利用者が確定額を集計欄に入力するのみで、実際の計算は別売の「freee人事労務」が担当）。個人事業主が一般従業員を雇うケース自体が限定的なため、aoikoも同じ方針とする。

### よくある仕訳パターン

| 取引 | 借方 | 貸方 |
|---|---|---|
| 売上をカードで受け取り | 1320 未収入金 | 4110 売上高 |
| 経費をクレカで払う | 5xxx 各経費 | 2120 未払金 |
| カード代金が銀行から引き落とし | 2120 未払金 | 1130 普通預金 |
| 現金を事業に投入 | 1110 現金 | 3110 元入金 |
| 生活費を事業口座から引出 | 1610 事業主貸 | 1130 普通預金 |
| 自宅家賃の事業分（家事按分 30%）| 5260 地代家賃 30% / 1610 事業主貸 70% | 1130 普通預金 |

## 2. 仕訳を確認する — 仕訳一覧

ナビ **「仕訳」** で `JournalList` 画面に移動。

### 2-1. 絞り込みフィルタ

画面上部のフィルタ群：

| フィルタ | 入力 |
|---|---|
| **年** | 数値（例：`2026`）|
| **月** | ドロップダウン（`全月` / `1`〜`12`）|
| **摘要に含む文字** | テキスト（部分一致、Enter または blur で適用）|
| **金額（下限）** | 数値、空欄＝無制限 |
| **金額（上限）** | 数値、空欄＝無制限 |
| **取引先** | ドロップダウン（登録済みの取引先から）|

> **2 つ以上の任意の組み合わせ**で絞り込めるため、優良な電子帳簿の検索要件を満たします（電子帳簿保存法 施行規則）。

**「リセット」** で全フィルタを初期化（年は今年、月は今月）。

### 2-2. 仕訳の詳細を見る

仕訳行をクリック → **展開**。借方／貸方の各明細（科目・補助科目・税率・金額・摘要）が表示されます。`▾` / `▸` のアイコンで展開状態を確認。

展開した明細の下に **証憑写真** の一覧（サムネイルクリックで原寸表示）と、写真を追加する欄があります。CSV 取込・OCR いずれで作成した仕訳でも、ここから後付けで証憑写真を補完できます（1-7 と同じ確認フローが挟まります）。

### 2-3. ページング

50 件ごとに分割。下部 **「← 前」** / **「次 →」** で移動。

## 3. 訂正仕訳（修正仕訳）— 間違いを正す

> aoiko は **確定済みの仕訳を直接編集できません**。電子帳簿保存法の要件（訂正履歴の保持）を満たすため、間違いは **打消し仕訳（reverse entry）** で訂正します。元の仕訳は履歴として残ります。

### 3-1. 訂正の手順

1. 仕訳一覧で該当の仕訳をクリックして展開
2. **「訂正する」** ボタンをクリック
3. 確認ダイアログ：「借方と貸方を入れ替えた打消し仕訳を本日付で作成し、元の仕訳は『訂正済み』となります」→ **「訂正する」**
4. 元の仕訳は **「訂正済み」** バッジ＆取り消し線で表示
5. 新しい打消し仕訳が一覧に追加されます（同日付）
6. 正しい内容で新規仕訳を入力（手動 / CSV / OCR / 注文取込のいずれかで）

### 3-2. 訂正のされ方の例

元の仕訳：
```
2026-05-01  電気代
  借方  5130 水道光熱費  8,000
  貸方  1130 普通預金    8,000
```

「訂正する」を押すと、本日付で打消し仕訳が自動生成：
```
2026-05-26  訂正：電気代
  借方  1130 普通預金    8,000
  貸方  5130 水道光熱費  8,000
```

これで原帳簿上、合計は 0。正しい金額（例：8,500）で新規仕訳を追加し直します。

### 3-3. 訂正の取り消しはできる？

一度訂正した仕訳に対し、さらに「訂正する」は押せません（**「訂正済み」** バッジが付き、ボタン非表示）。打消し仕訳側を訂正することで結果的に元に戻せますが、履歴は積み重なります。

> 大量のミスを修正したい場合は、CSV インポート由来なら **取込履歴**から **「バッチを全件訂正」** が便利（[03. CSV 取込](03-csv-import.md) 参照）。
