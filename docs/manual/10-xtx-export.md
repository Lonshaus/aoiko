# 10. `.xtx` 出力

e-Tax 形式の `.xtx` ファイルを生成し、e-Tax ソフト(WEB 版) で取込検証する。

**Language**: **日本語** | [English](10-xtx-export_en.md) | [繁體中文](10-xtx-export_zh-TW.md)

> **この章を読み終えたら出来ること**
> - 確定申告書（KOA020）+ 青色申告決算書一般用（KOA210）を 1 つの `.xtx` に出力できる
> - 出力された `.xtx` を e-Tax ソフト(WEB 版) で取込検証できる
> - 公式 XSD への準拠の意味と aoiko の責任範囲が理解できる
>
> **前提**：[01. 初回設定](01-setup.md) の基本情報、[09. 前期繰越](09-carryover.md) の期首振替、[08. 減価償却](08-depreciation.md) の年末償却 が完了し、当年度の仕訳が確定済みであること。

## 1. `.xtx` とは

`.xtx` は **e-Tax ソフト(WEB 版)** が受け付ける XML 形式の申告データファイル。aoiko は **国税庁公式 W3C XSD**（e-tax19 「XML スキーマ」由来）に従い、以下を 1 ファイルに併載した `.xtx` を生成します：

- **KOA020-023**：確定申告書 第一表〜第四表（令和 8 年分）
- **KOA210-011**：青色申告決算書（一般用）
- **手続コード**：`RKO0010`（所得税及び復興特別所得税申告）

aoiko の `.xtx` 構造は CI で **xmllint による公式 XSD validation** を毎回通しているため、構造的には公式準拠です。

## 2. aoiko の `.xtx` で何が出るか・出ないか

### 出力される内容

| 帳票 | 値の出所 |
|---|---|
| 確定申告書 第一表（年分、屋号）| Settings の基本情報 |
| 青色申告決算書 第 1 頁 損益計算書 | レポートの PL |
| 青色申告決算書 第 2 頁 月別売上（収入）金額 | レポートの月別売上 |
| 青色申告決算書 第 3 頁 減価償却費の計算 | 固定資産＋当年度償却 |
| 青色申告決算書 第 4 頁 貸借対照表 | レポートの BS |

### 出力されない内容（人手 or e-Tax 側で補完）

| 内容 | 補完方法 |
|---|---|
| 確定申告書本体の **個人情報**（氏名、住所、生年月日、個人番号）| e-Tax 側で入力 |
| **所得控除**（基礎控除、配偶者控除、社会保険料控除、医療費控除 等）| e-Tax 側で入力 |
| **税額計算**（所得税、復興特別所得税）| e-Tax 側で自動計算 |
| **添付資料**（医療費明細、寄付金控除証明 等）| e-Tax 側で添付 |
| **消費税申告書**（付表 2-3 等）| 別途 e-Tax 側で作成（aoiko は概算のみ。[07. 消費税](07-consumption-tax.md)）|

> aoiko の `.xtx` は **事業所得部分（決算書）+ 年分・屋号のみ**を埋めた骨格ファイル。残りは e-Tax 側で補完する前提です。

## 3. 出力手順

> aoiko のレポート画面または `.xtx` 出力 UI のメニュー（実装場所はバージョンにより異なる場合あり）から実行。

### 3-1. 出力前のチェック

1. 当年度の **仕訳がすべて確定済み**
2. **期首振替仕訳**（[09. 前期繰越](09-carryover.md)）が当年度の `2026-01-01` 付けで作成済み（初年度なら手動で作成済）
3. **減価償却仕訳**が年末に生成済み（[08. § 3](08-depreciation.md#3-年末に減価償却仕訳を生成)）
4. レポート → BS で **「資産合計」と「負債・純資産合計」が一致**（不整合警告が無い）
5. 設定 → 基本情報の **屋号、処理中の年度** が正しい

### 3-2. ファイル名

aoiko が生成するファイル名は `aoiko-{年度}.xtx` 等の形式（実装による）。ブラウザの「ダウンロード」フォルダに保存されます。

> aoiko の `.gitignore` には `aoiko-*.xtx` が含まれており、誤ってリポジトリにコミットされません。

## 4. e-Tax ソフト(WEB 版) での取込検証

**重要**：aoiko の `.xtx` は CI で公式 XSD validation を通していますが、**実機での取込検証は利用者責任**です。提出前に必ず以下を：

1. [e-Tax ホームページ](https://www.e-tax.nta.go.jp/) → e-Tax ソフト(WEB 版) にログイン
2. **「作成」 → 「ファイルから読込」** または同等メニュー
3. aoiko が出力した `.xtx` を選択して取込
4. 取込後、画面上で：
   - 決算書（一般用）の数値が aoiko のレポートと一致するか確認
   - 月別売上の各月が一致するか確認
   - 減価償却の各資産行が一致するか確認
   - 貸借対照表の各項目が一致するか確認
5. 不足項目（個人情報、所得控除、税額計算 等）を e-Tax 側で入力
6. **e-Tax 側でエラーが出る場合**：本ツールの出力に問題がある可能性があります。GitHub Issue で報告ください。

> **検証で問題が出た事例の対処**：CI の xmllint 通過 ≠ e-Tax 側で受け取れる、を意味します（schema 通っても e-Tax 側の業務ロジック側で弾かれる可能性は別途）。早い段階での実機検証推奨。

## 5. 内部仕様（参考）

### XSD ソース

- 配布元：国税庁 e-Tax「仕様公開資料」一覧
- 元 CAB：`e-tax19.CAB`「XML スキーマ」
- aoiko 同梱：`docs/xtx-spec/{shotoku,general}/` 配下の 7 ファイル
  - `KOA020-023.xsd`（確定申告書）
  - `KOA210-011.xsd`（青色申告決算書 一般用）
  - `General.xsd`、`ITdefinition.xsd`、`ITreference.xsd`、`zeimusho.xsd`、`zeimoku.xsd`

### 文書モデル

2 段式 ID/IDREF：

- **定義側**（IT 部）：`<IT VR="1.0" id="IT">` 内に値要素を ID 付きで列挙
- **参照側**（帳票個別部分）：各様式の leaf は IDREF で定義側を指す空要素

これは e-tax の標準モデル。aoiko の `.xtx` 生成スクリプトは XSD から自動生成された JSON スキーマを基に動的に組み立てます。

## 6. よくあるエラー

| エラー（e-Tax 側）| 推測される原因 |
|---|---|
| ファイル形式が違う | `.xtx` 以外、または別年度版を投入 |
| 様式が不一致 | aoiko の同梱 XSD が当年度版と乖離（[docs/xtx-spec/README.md](../../docs/xtx-spec/README.md) で最新版か確認）|
| 必須項目が不足 | 屋号未設定 等。Settings 基本情報を確認 |
| 数値の桁・形式が違う | 金額が負値、または非整数で入力されている等。仕訳を見直す |
