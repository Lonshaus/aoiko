# 10. `.xtx` 出力

e-Tax 形式の `.xtx` ファイルを生成し、e-Taxソフト(ダウンロード版) に組み込む。

**Language**: **日本語** | [English](10-xtx-export_en.md) | [繁體中文](10-xtx-export_zh-TW.md)

> **この章を読み終えたら出来ること**
> - 確定申告書（KOA020）+ 決算書（青色申告なら青色申告決算書一般用 KOA210、白色申告なら収支内訳書一般用 KOA110）を 1 つの `.xtx` に出力できる
> - 不動産所得を入力していれば、不動産所得用の決算書（青色：KOA220、白色：KOA130）も同じ `.xtx` に併載される（[14. 所得控除・税額控除](14-income-deductions.md) 参照）
> - 出力された `.xtx` を e-Taxソフト(ダウンロード版) に組み込める
> - aoiko が埋める範囲（事業。[14. 所得控除・税額控除](14-income-deductions.md) を入力済みなら所得控除・税額も）と利用者が e-Tax 側で補完する範囲が理解できる
>
> **前提**：[01. 初回設定](01-setup.md) の基本情報・申告者情報、[09. 前期繰越](09-carryover.md) の期首振替、[08. 減価償却](08-depreciation.md) の年末償却 が完了し、当年度の仕訳が確定済みであること。

## 1. `.xtx` とは

`.xtx` は **e-Taxソフト(ダウンロード版)** が「組み込み」で受け付ける XML 形式の申告データファイル。aoiko は **国税庁公式 W3C XSD**（e-tax19「XML スキーマ」由来）に従い、以下を 1 ファイルに併載した `.xtx` を生成します：

- **KOA020-023**：確定申告書 第一表
- **KOA210-011**：青色申告決算書（一般用）、または **KOA110-012**：収支内訳書（一般用）——**設定 → 申告者情報 → 確定申告方式**（青色申告／白色申告）でどちらを同梱するか決まる
- **KOA220-008**：青色申告決算書（不動産所得用）、または **KOA130-009**：収支内訳書（不動産所得用）——**設定 → 不動産所得を使う** を有効にし、[14. 所得控除・税額控除](14-income-deductions.md) で不動産所得を入力している場合のみ、上記の事業用決算書に加えて同梱される
- **TEA060**：申告書等送信票（兼送付書）
- **手続コード**：`RKO0010`（所得税及び復興特別所得税申告）

aoiko の `.xtx` は CI で **xmllint による公式 XSD validation** を通し、かつ e-Taxソフト(ダウンロード版) での **実機組み込み検証済み**です。

> **不動産所得の貸倒金・貸倒引当金繰入額について**：青色申告決算書（不動産所得用 KOA220）にはこの2科目専用の欄が無いため、様式が用意している「追加科目」欄（上限5件）へ出力します。収支内訳書（不動産所得用 KOA130）には「貸倒金」専用欄があるためそちらに直接出力しますが、白色申告に引当金の制度自体が無いため「貸倒引当金繰入額（不動産）」はKOA130には出力されません（詳細は [14. 所得控除・税額控除](14-income-deductions.md) 参照）。

> **e-Taxソフト(WEB 版) は所得税申告(RKO0010)の組み込みに非対応**です。必ず **ダウンロード版**（税目別モジュールのインストールが必要）を使用してください。作成コーナーは `.xtx` の組み込みには対応していません。

## 2. aoiko の `.xtx` で何が出るか・出ないか

aoiko は **事業の損益** を扱うツールです。`.xtx` には決算書と、確定申告書の **事業部分**＋提出に必須の **申告者情報** までが入ります。

### 出力される内容

| 帳票 | 値の出所 |
|---|---|
| 申告者情報（税務署・利用者識別番号・氏名・住所）| Settings の申告者情報 |
| 確定申告書 第一表：営業等収入金額（ア）| レポートの PL（売上） |
| 確定申告書 第一表：事業所得（①）| PL（白色申告は経費控除後の額。青色申告はさらに青色申告特別控除を差し引いた額）|

**青色申告の場合**（決算書：青色申告決算書一般用 KOA210）

| 帳票 | 値の出所 |
|---|---|
| 確定申告書 第一表：青色申告特別控除額 | PL ＋ 青色控除区分 |
| 決算書 第 1 頁 損益計算書（青色控除前/控除額/控除後 含む）| レポートの PL ＋ 青色控除区分 |
| 決算書 第 2 頁 月別売上（収入）金額・仕入金額 | レポートの月別（仕入は売上原価・仕入のみ）|
| 決算書 第 4 頁 貸借対照表 | レポートの BS |

**白色申告の場合**（決算書：収支内訳書一般用 KOA110）

| 帳票 | 値の出所 |
|---|---|
| 収支内訳書 第 1 頁 収入金額・経費各科目・専従者控除前の所得金額 | レポートの PL |

> 白色申告には貸借対照表・月別売上の対映欄がありません（収支内訳書にその項目自体が無いため）。専従者給与・貸倒引当金繰入額は収支内訳書に対応欄が無く、所得計算からも除外されます（下記「e-Tax 側で補完する内容」参照）。

### e-Tax 側で補完する内容

| 内容 | 補完方法 |
|---|---|
| **各種所得控除**（基礎控除、配偶者控除、社会保険料控除、医療費控除 等）| [14. 所得控除・税額控除](14-income-deductions.md) で入力済みなら自動出力。未入力なら e-Tax 側で入力 |
| **合計所得金額・課税所得・税額計算**（所得税、復興特別所得税）| 同上（所得控除の入力があれば aoiko が計算して出力。未入力なら e-Tax 側で自動計算）|
| **添付資料**（医療費明細、各種控除証明 等）| e-Tax 側で添付 |
| **消費税申告書**（複数事業区分の簡易課税・3 割特例）| 別途 e-Tax 側で作成（aoiko は概算のみ。本則課税・2 割特例・簡易課税〔単一事業区分〕は `.xtx` 出力対応：[§ 6](#6-消費税本則課税2-割特例簡易課税の-xtx-出力)、[07. 消費税](07-consumption-tax.md)）|
| **白色申告の専従者控除**（配偶者86万円・その他親族50万円の定額控除）と控除後の所得 | e-Tax 側で入力（続柄情報が必要なため aoiko では算定しない）|

> 記帳・決算書・事業所得まではソフトが作り、残りは確定申告の段階で本人が e-Tax 上で補完する——という会計ソフト共通の役割分担が基本です。aoiko は [14. 所得控除・税額控除](14-income-deductions.md) の入力があれば、所得控除・税額まで出力します。

## 3. 出力手順

### 3-1. 出力前のチェック

1. **設定 → 申告者情報** に 税務署・利用者識別番号・氏名・住所 を入力済み（未入力だと `.xtx` を書き出せません）
2. **設定 → 確定申告方式** が実際の申告方式（青色申告／白色申告）と一致している
3. 青色申告の場合、**設定 → 青色申告特別控除** の区分（65万/55万/10万）が正しい
4. 当年度の **仕訳がすべて確定済み**
5. **期首振替仕訳**（[09. 前期繰越](09-carryover.md)）が当年度の `2026-01-01` 付けで作成済み（初年度なら手動で作成済）
6. **減価償却仕訳**が年末に生成済み（[08. § 3](08-depreciation.md#3-年末に減価償却仕訳を生成)）
7. 青色申告の場合、レポート → BS で **「資産合計」と「負債・純資産合計」が一致**（不整合警告が無い）。白色申告の収支内訳書には貸借対照表が無いためこの確認は不要

### 3-2. ファイル名

aoiko が生成するファイル名は `aoiko-{年度}.xtx` 形式。ブラウザの「ダウンロード」フォルダに保存されます。

> aoiko の `.gitignore` には `aoiko-*.xtx` が含まれており、誤ってリポジトリにコミットされません。

> **令和8年分の取扱い**：令和8年分（2026 年分）の e-Tax 所得税モジュールは申告期（2027 年）まで提供されません。動作確認は、現在提供中の令和7年分モジュールに対し、開発ビルドの「テスト: 令和7年分で書き出し」で行えます（封包構造は年分に依存しません）。

## 4. e-Taxソフト(ダウンロード版) での組み込み

**重要**：aoiko の `.xtx` は CI と実機で検証していますが、**実際の申告内容の最終確認は利用者責任**です。

1. e-Taxソフト(ダウンロード版) を起動し、**所得税の年分モジュールを追加インストール**（未導入の場合）
2. **利用者を選択** → 「**申告・申請等**」
3. 「**組み込み**」 → aoiko が出力した `.xtx` を選択
4. 3 帳票（確定申告書・決算書〔青色申告決算書 または 収支内訳書〕・送信票）が「組み込み」状態になることを確認
5. 「**帳票編集**」 で数値を確認：
   - 青色申告：決算書（一般用）の損益・月別・貸借対照表が aoiko のレポートと一致するか
   - 白色申告：収支内訳書（一般用）の収入・経費各科目・専従者控除前の所得金額が aoiko のレポートと一致するか
   - 確定申告書 第一表の 営業等収入・事業所得（青色申告のみ・青色申告特別控除額）・氏名・住所・税務署 が正しいか
6. 不足項目を e-Tax 側で入力（[14. 所得控除・税額控除](14-income-deductions.md) を入力済みなら所得控除・税額は出力済み。未入力の所得控除、白色申告の専従者控除 等が対象）
7. **組み込み時にエラーが出る場合**：本ツールの出力に問題がある可能性があります。GitHub Issue で報告ください。

## 5. 内部仕様（参考）

### XSD ソース

- 配布元：国税庁 e-Tax「仕様公開資料」一覧
- 元 CAB：`e-tax19.CAB`「XML スキーマ」
- aoiko 同梱：`docs/xtx-spec/{shotoku,general}/` 配下
  - `KOA020-023.xsd`（確定申告書）・`KOA210-011.xsd`（青色申告決算書 一般用）・`KOA110-012.xsd`（収支内訳書 一般用）
  - `KOA220-008.xsd`（青色申告決算書 不動産所得用）・`KOA130-009.xsd`（収支内訳書 不動産所得用）
  - `General.xsd`、`ITdefinition.xsd`、`ITreference.xsd`、`zeimusho.xsd`、`zeimoku.xsd`

### 文書モデル

2 段式 ID/IDREF：

- **エンベロープ**：`<DATA>`（5 名前空間）→ 手続要素 `RKO0010` → `CATALOG`（RDF マニフェスト）＋ `CONTENTS`
- **定義側**（IT 部）：`<IT VR="1.5" id="IT">` 内に申告者情報・年分等を ID 付きで列挙
- **参照側**（帳票個別部分）：各様式は単一様式要素がページ子要素を内包し、leaf は IDREF で定義側を指す
- **送信票**：`SOFUSHO`（TEA060、kyotsu 名前空間）

aoiko の `.xtx` 生成は XSD から自動生成した JSON スキーマを基に動的に組み立てます。

## 6. 消費税（本則課税・2 割特例・簡易課税）の `.xtx` 出力

上記の所得税 `.xtx`（手続 `RKO0010`）とは **別ファイル・別手続** です。消費税の課税期間（暦年）が確定してから、所得税とは別に組み込みます。

### 対応範囲

- **手続 `RSH0010`**（消費税及び地方消費税申告）で、以下いずれかを出力：
  - **本則課税**：消費税及び地方消費税の申告書（一般用）＋ 付表 1-3 ＋ 付表 2-3。**課税売上割合 100%（非課税売上・輸出免税等が無い）を前提**とします（付表 2-3 は「課税売上高5億円以下・課税売上割合95%以上」の全額控除区分のみ使用。個別対応方式・一括比例配分方式は未対応）
  - **2 割特例**：消費税及び地方消費税の申告書（簡易課税用）＋ 付表 6（税額控除に係る経過措置＝2 割特例）。2 割特例は簡易課税を正式に選択していない事業者もこの様式構造を使う運用です（国税庁の 2 割特例専用手引きに基づく）
  - **簡易課税（単一事業区分のみ）**：消費税及び地方消費税の申告書（簡易課税用）＋ 付表 4-3 ＋ 付表 5-3
- **対応しているのは本則課税・2 割特例・簡易課税（単一事業区分）のみ**。複数事業区分にまたがる簡易課税（75% ルール等）・3 割特例の申告書出力は未対応（[07. 消費税](07-consumption-tax.md) 参照）
- **既知の簡略化**：売上対価の返還等（返品・値引）は、aoiko の集計で課税標準額へネット計上済みのため、付表の該当欄には内訳表示されません。**最終税額自体は正しく計算されます**

### 出力手順

1. **設定 → 消費税** で「課税事業者」に設定（簡易課税を出力する場合はさらに **「簡易課税」＋事業区分** も設定）
2. **設定 → 申告者情報** に 税務署・利用者識別番号・氏名・住所 を入力済み
3. レポート画面 → 「**消費税**」 セクションで対象の方式が方式一覧に表示されていることを確認
4. 「**本則課税の `.xtx` をダウンロード**」・「**2 割特例の `.xtx` をダウンロード**」・「**簡易課税の `.xtx` をダウンロード**」 のいずれかをクリック（ファイル名 `aoiko-shohi-{年度}.xtx`）

### e-Taxソフト(ダウンロード版) での組み込み

所得税とは別に、**消費税の年分モジュール**を追加インストールしてから組み込みます。手順は [§ 4](#4-e-taxソフトダウンロード版-での組み込み) と同様（「組み込み」→ ファイル選択 → 帳票編集で数値確認）。

## 7. よくあるエラー

| エラー（e-Tax 側）| 推測される原因 |
|---|---|
| `SC00X010 このファイルを組み込むことはできません` | 申告者情報の未入力、または別年度モジュールへの投入 |
| ファイル形式が違う | `.xtx` 以外を投入 |
| 様式が不一致 | aoiko の同梱 XSD が当年度版と乖離（[docs/xtx-spec/README.md](../../docs/xtx-spec/README.md) で最新版か確認）|
| 数値の桁・形式が違う | 金額が負値、または非整数で入力されている等。仕訳を見直す |
