aoiko 是輔助日本個人事業主青色申告・白色申告作業的工具。使用本工具即視為同意以下各點。

- 確定申告・修正申告・稅務調查對應時發生的任何損害（追徵課税、加算税、延滯税、過少申告加算税、其他），開發者**一概不負責任**。
- 最終數字的妥當性請務必跟**税理士・税務署確認**。

## 2. 稅法・帳簿要件每年都變

- 勘定科目・税率・控除額・帳簿保存要件**每年都改正**。
- 本工具同梱的 `tax-schema/{年度}` 是以**當時的理解**製作的，可能跟最新的国税庁公告有差異。
- 使用前請確認當下年度的 `tax-schema/` 與最新版一致。

## 3. `.xtx` 輸出與實機驗證

- `.xtx` 輸出採用国税庁官方 W3C XSD（來源 e-tax19「XML スキーマ」）的 2 段式 ID/IDREF 文件模型生成，將確定申告書（KOA020）+「設定 ＞ 申告者資訊」裡確定申告方式對應的決算書（青色申告決算書一般用 KOA210，或白色申告用的収支内訳書一般用 KOA110）以單一送信資料（手続 `RKO0010` 所得税及び復興特別所得税申告）併載。各様式參照側 CI 已通過官方 xsd validation（xmllint）。
- 申告書側會帶入申報者資訊（稅務署・利用者識別號碼・姓名・地址）與事業的 **營業等收入・事業所得**（僅青色申告會加上 **青色申告特別控除**）。**各項所得控除與稅額計算不在 aoiko 範圍內**,請在 e-Tax 上自行補完。決算書側對應損益計算書・資產負債表・月別銷貨（仕入），**白色申告的収支内訳書不含資產負債表**。
- **白色申告的専従者控除**（配偶者86萬圓・其他親屬每人50萬圓的定額控除）因需要續柄資訊，aoiko 不會自動算定。只會輸出専従者控除前的所得金額，**控除額與控除後所得請務必自行在 e-Tax 上輸入**。
- **實申告使用前請務必用 e-Tax 軟體(下載版) 組み込み並確認內容**（所得稅的組み込み WEB 版不支援）。手續代碼・附件要件・各項控除等隨使用者狀況而異。本工具輸出直接提交所產生的任何結果開發者一概不負責（依第 1 條）。

## 3a. 消費税申告書不在範圍

- 本工具提供的消費税計算**僅限概算與方式比較**（本則・簡易・2 割特例・3 割特例）。
- **消費税申告書本體（消費税申告書 ＋ 付表 2-3 等）的輸出不在範圍**。實申告請走確定申告書等作成コーナー或税理士。
- 仕入税額控除的経過措置（80/70/50/30%）與簡易課税みなし仕入率以工具內建表判定，但特殊情況（混合事業・控除對象外仕入・調整税額等）未對應。

## 4. 電子帳簿保存法・發票制度的遵循是利用者責任

- 本工具**不保證完全符合電子帳簿保存法的 scanner 保存要件・電子取引要件**（時間戳・檢索機能・訂正履歷僅部分實裝）。
- 適格請求書（發票）真實性（T 番号有効性比對等）請**利用者自行於国税庁公開網站等確認**。本工具把傳票標記為「インボイス対応」不代表法律上有效。

## 5. LLM / OCR 機能風險

- LLM 分類・OCR 引擎可在設定中選擇。
  - **Google Gemini（預設・雲端）**：送出內容（CSV 行、收據圖片）依 **Google 隱私政策**與利用者 API 方案合約處理（免費額度可能被用於訓練）。
  - **OpenAI 相容 / Ollama 等（本地）**：endpoint 是 localhost 時資料不離開本機。指定遠端時依該服務的政策。
  - **Tesseract（純本地 WASM OCR・OCR 專用）**：影像不離開本機。不用 LLM、只從 OCR 文字以確定性規則抽取 T+13 位登錄番号・日期・合計。店名與品項不推測。**精度比其他引擎差很多**，利用者務必人工確認與修正。首次會從 CDN 取得 `jpn.traineddata` / `eng.traineddata`（設定可自架做到完全離線）。
- 外部（雲端）送出引擎使用時送出前會跳確認對話框。可能含機微資訊・第三方個資的資料送出前請務必確認內容。
- 本地 AI（Ollama 等）**OCR 必須選 vision 對應模型**。aoiko 自己也要本機執行（HTTPS 配信版不能連 localhost），Ollama 那邊也要設 `OLLAMA_ORIGINS`。
- LLM 輸出**可能有錯誤**。確定前務必人工驗證。
