## 2. 税法・帳簿要件は年度ごとに変わる

- 勘定科目・税率・控除額・帳簿保存要件は **毎年改正されます**。
- 本ツールに同梱されている `tax-schema/{年度}` は **その時点の理解** に基づいて作られており、最新の国税庁公告と乖離している可能性があります。
- 利用前に、現在の年度の `tax-schema/` が最新版と一致しているか確認してください。

## 3. `.xtx` 出力と実機検証

- `.xtx` 出力は国税庁公式 W3C XSD（e-tax19「XMLスキーマ」由来）の 2 段式
  ID/IDREF 文書モデルで生成し、確定申告書（KOA020）+ 青色申告決算書一般用
  （KOA210）を 1 つの送信データ（手続 `RKO0010` 所得税及び復興特別所得税申告）に
  併載します。各様式の参照側は公式 xsd validation（xmllint）を CI で通しています。
- 申告書側には 申告者情報（税務署・利用者識別番号・氏名・住所）と、事業の
  **営業等収入・事業所得・青色申告特別控除** が入ります。**各種所得控除・税額計算は
  aoiko の対象外**で、e-Tax 上で本人が補完してください。決算書側は損益計算書・
  貸借対照表・月別売上（仕入）を対映します。
- **実申告に使用する前に、必ず e-Taxソフト(ダウンロード版) で組み込み・内容確認を
  行ってください**（所得税の組み込みは WEB 版 非対応）。手続コード・添付要件・各種
  控除等は利用者の状況により異なります。本ツール出力をそのまま提出して生じた
  いかなる結果についても開発者は責任を負いません（第 1 項）。

## 3a. 消費税申告書は対象外

- 本ツールが提供する消費税計算は **概算と方式比較（本則・簡易・2 割特例・3 割特例）に限定** されます。
- **消費税申告書本体（消費税申告書 + 付表 2-3 等）の出力は対象外** です。実申告は確定申告書等作成コーナーまたは税理士に依頼してください。
- 仕入税額控除の経過措置（80/70/50/30%）と簡易課税のみなし仕入率はツール内蔵テーブルで判定しますが、特殊なケース（混合事業・控除対象外仕入・調整税額等）には対応していません。

## 5. LLM / OCR 機能のリスク

- LLM 分類・OCR のエンジンは設定で選べます。
  - **Google Gemini（既定・クラウド）**：送信内容（CSV 行、領収書画像）は **Google のプライバシーポリシー** および利用者の API プラン契約に従って取り扱われます（無料枠は学習利用される場合あり）。
  - **OpenAI 互換 / Ollama 等（ローカル）**：エンドポイントが localhost の場合データは端末外に出ません。リモートを指定した場合は当該サービスのポリシーに従います。
  - **Tesseract（純ローカル WASM OCR・OCR 専用）**：画像は端末外に出ません。LLM を使わず OCR テキストから T+13 桁登録番号・日付・合計を確定性ベースで抽出するのみ。店名・品目は推定しません。**精度は他エンジンに比べ大きく劣る**ため、利用者が必ず人手で確認・修正してください。初回のみ `jpn.traineddata` / `eng.traineddata` を CDN から取得します（設定で自己ホスト可能・完全オフライン運用可）。
- 外部（クラウド）送信エンジン使用時は送信直前に確認ダイアログを表示します。機微情報・第三者の個人情報を含む可能性のあるデータの送信前に、必ず内容を確認してください。
- ローカル AI（Ollama 等）は **OCR には vision 対応モデルが必須** です。また aoiko をローカル実行（HTTPS 配信版は localhost 不可）し、Ollama 側で `OLLAMA_ORIGINS` の設定が必要です。
- LLM の出力は **誤りを含む可能性** があります。確定前に必ず人間が検証してください。
